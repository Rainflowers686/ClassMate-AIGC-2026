# Stage 7 Demo Narration Long

## 0:00-0:30 首页：项目定位

现在展示的是 ClassMate。它不是普通录音总结工具，而是一个课堂学习助手。我们关注的是“课堂资料进入系统之后，学生怎么复习、怎么练习、怎么验证来源”。所以主链路是：导入资料，生成知识点，查看证据链，完成微测，进入复习和需要多练，最后导出学习报告。

当前页面是首页，能看到今日学习状态、导入课堂资料、Live Companion、最近一节课和模型状态。首页刻意保持简洁，不把所有功能都堆在第一屏。

## 0:30-1:30 Settings：官方路径和安全边界

这里进入设置页。参赛合规主路径是 Official BlueLM/qwen3.5-plus。Compatible Demo 只用于展示兼容增强，LocalFallback 是兜底路径，保证网络或模型不可用时还能展示基本学习流程。

安全方面，本地凭据不进入仓库、日志、历史、导出和截图。日志只展示短状态，比如 provider、耗时和错误类型，不展示内部模型交互。这里也能看到能力路线图：已支持、实验模式、待接入和暂缓。我们不会把未完成能力包装成已完成。

## 1:30-3:00 Import：多来源导入

现在进入导入课堂资料。复赛版支持粘贴文本、Markdown、示例课堂、手动 OCR 资料流、SRT/VTT/TXT 字幕或转写稿。手动 OCR 适合先粘贴课件、PPT、板书或讲义识别出来的文字；真实 vivo OCR provider 是后续接入方向。

进入资料篮后，可以看到不同来源进入同一个 MaterialBundle。这里的价值是：分析器不需要知道资料来自文本、字幕还是 OCR，它统一处理为课堂资料，并保留来源 marker。

## 3:00-4:00 Live：手动课堂和 ASR 实验

现在展示 Live Companion。复赛版支持手动课堂片段，也有系统 ASR 实验模式。实验模式依赖设备系统 SpeechRecognizer，不保存原始音频，不后台录音。这里要强调：我们没有声称真实 vivo ASR provider 已经接入，也不做声纹身份识别。

如果设备环境不稳定，仍然可以用手动片段或导入转写稿完成同样的学习闭环。

## 4:00-5:30 Timeline / Evidence：知识时间线和证据链

现在生成知识时间线。ClassMate 不希望模型只输出一段摘要，而是把课堂资料拆成知识点。每个知识点都有标题、难度、重要性和证据引用。

打开证据详情，可以看到 evidence quote 和来源 marker。这个来源可能是课堂文本、字幕、课件 OCR、板书 OCR 或 Live 片段。证据链是 ClassMate 的核心，因为它让学生知道知识点来自哪里，也让评委能验证系统没有随意编造。

## 5:30-6:40 Ask This Lesson：基于本节课证据提问

接下来是 Ask This Lesson。它不是通用聊天，而是基于本节课资料回答。我们准备三类问题：第一类有明确证据，系统应该回答 grounded；第二类只有部分依据，系统应该回答 partial；第三类本节课没有依据，系统应该回答 not_found。

这能展示防胡编机制：当资料中没有明确依据时，系统应该拒绝强答，而不是编一个看似正确的答案。

## 6:40-8:00 Quiz / Review / 需要多练

然后进入微测。微测题围绕已提取知识点生成，答题后会显示解释和证据。答错、太难、需要多练、证据存疑等反馈会进入学习状态。

Review 页面展示今日任务、薄弱点和优先级。需要多练不是一个单独标签，而是连接到练习搜索和后续 Adaptive Practice 的入口。练习搜索只打开外部搜索结果，不抓取第三方平台内容。

## 8:00-9:10 Course Library / Export Center

课程库把历史记录按课程聚合，支持搜索、筛选和排序。点击课程可以回到课程详情，继续看时间线、Ask、Quiz、Review 和资料来源。

最后打开 Export Center。它解决了普通用户找不到导出文件的问题，可以保存到用户选择的位置，也可以走系统分享。支持 PDF、HTML、Markdown、TXT、MindMap、Word 兼容 HTML 和 Slides HTML。这里的 Word 和 Slides 是兼容 HTML，不是原生 Office 文件。

## 9:10-10:00 收尾：对比和路线

和录音转写类产品相比，ClassMate 也在补齐输入能力，但差异化是学习闭环。录音转写解决“记下来”，ClassMate 继续解决“学会它”：证据链、微测、复习、需要多练、课程库和报告导出。

诚实边界也很重要：不做第三方平台视频爬取，不伪装真实 vivo ASR/OCR provider 已完成，声纹身份识别和云同步暂缓。决赛版会把真实 ASR/OCR provider 接入现有 MaterialBundle 架构，继续增强多来源证据和学习画像。
