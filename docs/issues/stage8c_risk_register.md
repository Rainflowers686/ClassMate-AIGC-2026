# Stage 8C Risk Register

> 风险台账，覆盖端侧模型、CourseAnalysis JSON、validators、图片/拍照链路、权限、真机兼容、导出报告、安全/密钥、评委质疑和时间/期末周十大类别，共计 45 条。

---

## 一、端侧模型风险（7 条）

| # | 风险 | 触发条件 | 影响 | 当前缓解措施 | Owner | 优先级 |
|---|------|----------|------|-------------|-------|--------|
| R01 | 端侧 SDK init 失败 | native library 加载异常、ABI 不匹配、设备不支持 | 端侧 fallback 不可用，双模型失败走安全占位 | AAR 只包含 arm64-v8a；Settings 诊断页预检；真机 X300 Pro 已验证 | SDK 对接人 | P0 |
| R02 | 端侧模型文件缺失或损坏 | `/sdcard/1225` 目录无模型文件或文件校验失败 | 端侧推理无法执行 | Settings 诊断显示 model path 状态；导入前路径可访问性检查 | 设备管理员 | P0 |
| R03 | 端侧 generate 超时 | 长文本输入 + 设备性能不足 + 模型推理慢 | 课程分析超时，回退安全占位 | 端侧 prompt 控制输入长度；超时阈值设置和 fallback 链路 | SDK 对接人 | P1 |
| R04 | 端侧推理内存不足 | 3B 模型加载 + 推理同时占用内存，低 RAM 设备 OOM | App 崩溃或 generate onError | Settings 诊断预检；release() 生命周期管理 | SDK 对接人 | P1 |
| R05 | 端侧 NPU/APU 不可用 | 设备缺少 NPU 或 NPU 驱动不兼容 | 推理回落 CPU，速度显著降低 | 文档诚实说明端侧性能依赖设备算力 | SDK 对接人 | P2 |
| R06 | 端侧 generate 返回乱码/截断 | 模型输出超出 maxTokens 或解码异常 | 输出 JSON parse 失败，不落库 | validators JSON parse guard；截断检测 | SDK 对接人 | P1 |
| R07 | 端侧多模态 callVit 失败 | 图片尺寸异常、RGB 转换失败、VIT 模型未加载 | 图片草稿无法生成，需用户手动输入 | callVit 诊断预检；失败时提示用户手动输入文字 | SDK 对接人 | P1 |

## 二、CourseAnalysis JSON 风险（5 条）

| # | 风险 | 触发条件 | 影响 | 当前缓解措施 | Owner | 优先级 |
|---|------|----------|------|-------------|-------|--------|
| R08 | 端侧 CourseAnalysis 输出非 JSON | 模型未遵循 prompt JSON 格式要求 | parse 失败，知识点不落库 | JSON parse try/catch；parse 失败走安全占位 | 后端/AI 对接人 | P0 |
| R09 | 端侧 CourseAnalysis JSON 字段缺失 | 模型输出 JSON 缺少 title/importance/evidence 等必要字段 | 知识点不完整，落库拦截 | validators schema 校验；缺失字段的兜底默认值策略 | 后端/AI 对接人 | P0 |
| R10 | 端侧 CourseAnalysis 输出知识点数量为 0 | 模型无法从输入提取有效知识点 | 课程分析空结果，用户体验差 | 空知识点检测 → 安全占位提示 | 后端/AI 对接人 | P1 |
| R11 | 云端和端侧 JSON schema 不一致 | 端侧 prompt 与云端 prompt 产出格式差异 | validators 对两端校验结果不一致 | 统一 validators；端侧 prompt 明确输出 schema | 后端/AI 对接人 | P1 |
| R12 | CourseAnalysis JSON 包含不允许字段（如 reasoning） | 模型输出携带 reasoning_content 或内部推理 | 违反安全边界 | redaction 机制移除敏感字段 | 后端/AI 对接人 | P2 |

## 三、validators 风险（5 条）

| # | 风险 | 触发条件 | 影响 | 当前缓解措施 | Owner | 优先级 |
|---|------|----------|------|-------------|-------|--------|
| R13 | validators 漏过不合格知识点 | 校验规则覆盖不全，伪造成立的知识点通过 | 假知识点入知识库，评委可发现 | 多层校验（JSON + schema + evidence + 范围）；red team review | 后端/AI 对接人 | P0 |
| R14 | validators 误拦合格知识点 | 边界条件过于严格，正常知识点被拒绝 | 知识点损失，分析不完整 | 校验阈值调节；validators 日志审计 | 后端/AI 对接人 | P1 |
| R15 | validators 对 evidence 关联校验不充分 | 知识点声称有证据但其实不匹配 | 虚假证据链，损害可信度 | evidence quote 匹配度阈值；spot check | 后端/AI 对接人 | P0 |
| R16 | validators 性能瓶颈 | 大批量知识点校验耗时过长 | 课程分析总耗时增加，用户等待 | 校验逻辑轻量化；单知识点校验 < 50ms | 后端/AI 对接人 | P2 |
| R17 | validators 未适配端侧输出质量 | 端侧输出格式与云端细微差异导致校验失败率偏高 | 端侧分析通过率低，频繁回退安全占位 | 端侧输出专项适配；端侧 tolerant 校验模式 | 后端/AI 对接人 | P1 |

| R22 | 大尺寸图片导致 VIT 编码 OOM | 高分辨率照片 RGB 数据量过大 | callVit 失败或 App 崩溃 | 图片预处理缩放；尺寸上限检查 | SDK 对接人 | P1 |
| R23 | 图片导入入口不明显 | 用户不知道可以拍照/选图导入 | 端侧多模态功能使用率低 | Import Hub 设计突出图片入口 | UX 对接人 | P2 |
