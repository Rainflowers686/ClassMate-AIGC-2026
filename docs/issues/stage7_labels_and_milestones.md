# Stage 7 Labels And Milestones

## Labels

| Label | 用途 |
|---|---|
| `p0` | 复赛阻塞或必须当天解决 |
| `p1` | 复赛前优先完成 |
| `p2` | 可延后但影响质量 |
| `p3` | 决赛或长期研究 |
| `bug` | 缺陷 |
| `feature` | 新功能 |
| `docs` | 文档 |
| `test` | 测试与验收 |
| `security` | 安全与隐私 |
| `ui` | UI/UX |
| `provider` | 模型/provider 接入 |
| `review-loop` | 复习、错题、需要多练 |
| `export` | Export Center / StudyReport |
| `asr` | ASR / 转写 |
| `ocr` | OCR / 课件 / 板书 |
| `proof` | 截图、录屏、proof pack |
| `competition` | 复赛/答辩提交 |

## Milestones

### Stage 7 Stabilization

目标：修复当前 Stage 7C 编译与真机阻塞，保证 APK 可安装、主链路可演示。

建议包含：

- Review/Practice compile blockers。
- BlueLM long lesson smoke。
- Export Center smoke。
- Secrets scan。

### Rematch Demo

目标：复赛演示视频、截图、问答和提交包。

建议包含：

- Demo script。
- Proof screenshot list。
- Judge Q&A。
- Submission package checklist。

### Finalist Foundation

目标：决赛地基，继续完善学习闭环和产品成熟度。

建议包含：

- Course Library polish。
- Adaptive Practice stabilization。
- StudyReport polish。
- UI overflow fix。

### ASR/OCR Provider Integration

目标：真实 provider 接入前的架构、文档、风险和后续实现。

建议包含：

- vivo ASR provider design。
- vivo OCR provider design。
- MaterialBundle evidence markers。
- Transcript/OCR test assets。

### UI Polish

目标：Focus 主视觉稳定，Flow 局部体验，移动端小屏可用。

建议包含：

- Focus theme pass。
- Small-screen overflow。
- Settings roadmap clarity。
- Course Detail proof polish。
