# Ask This Lesson 复赛演示脚本

## 1. 为什么不是普通 AI 问答

“问这节课”不是把问题直接丢给通用 AI。ClassMate 的目标是回答“这节课里讲过什么”，所以系统必须先看当前课堂的知识点、证据片段和课程标题，再决定能不能回答。

普通问答可以凭常识补全很多内容，但课堂学习场景里，答案必须能追溯到本节课材料。这样做可以避免学生把模型的课外发挥误当成老师今天讲过的内容。

## 2. 为什么必须基于本节课证据

ClassMate 的核心不是摘要，而是“课堂内容 -> 可验证知识点 -> 微测 -> 复习任务 -> 反馈优化”。如果 Ask This Lesson 没有证据约束，就会破坏这条链路。

演示时要强调三点：

- 答案应标注 grounded / partial / not_found。
- grounded 答案要显示相关证据。
- 没有证据时系统应拒绝胡编，而不是给出看似完整的课外答案。

## 3. 演示问题 1：直接有证据

推荐课程：高等数学：数项级数收敛判别法。

操作：

1. 在 Import 中加载高数样例课堂。
2. 生成知识时间线。
3. 进入 Ask This Lesson。
4. 输入：“通项 an 趋于 0 是否能保证级数收敛？”

期望展示：

- 状态：grounded。
- 回答：不能保证；通项趋于 0 是必要条件，不是充分条件。
- 证据：能看到“必要条件”“不是充分条件”“调和级数”等相关证据词。

讲解词：

“这个问题在课堂文本里有直接依据。系统不是泛泛解释级数，而是引用本节课里关于必要条件和调和级数反例的证据。”

## 4. 演示问题 2：部分依据

推荐课程：大学物理：电磁感应。

问题：“线圈方向不垂直时如何计算动生电动势？”

期望展示：

- 状态：partial。
- 回答：本节课提到方向不垂直时要取有效分量，但没有展开完整向量公式。
- 证据：能看到“有效分量”“方向不垂直”相关证据。

讲解词：

“这个问题不是完全没有依据，但课堂只讲到有效分量，没有完整推导。所以系统应给出部分回答，并明确边界。”

## 5. 演示问题 3：本节课没有依据

推荐课程：AI/机器学习：机器学习基本流程。

问题：“Transformer 的自注意力机制怎么计算？”

期望展示：

- 状态：not_found。
- 回答：本节课资料中没有找到明确依据。
- 不应输出自注意力公式或大段课外解释。

讲解词：

“这个问题是 AI 领域相关问题，但不是这节课讲的内容。ClassMate 应拒绝把课外知识冒充为课堂证据，这就是和普通随时问答的关键区别。”

## 6. 展示引用证据

演示时在 grounded 或 partial 答案下方停留 5-8 秒，展示：

- related knowledge point；
- evidence quote；
- segment 或片段来源；
- groundedness 状态。

讲解词：

“评委可以看到，答案不是孤立生成的，它和知识点、证据片段、后续微测和复习任务是连在一起的。”

## 7. 与录音转写类产品“随时问”的差异

录音转写类产品的“随时问”更偏通用问答、总结和会议助手。ClassMate 借鉴这种交互，但把它限制在课堂证据链里。

差异点：

- 听课材料不是只用于总结，还会变成知识点、微测和复习任务。
- Ask This Lesson 需要显示 grounded / partial / not_found。
- 没有证据时不胡编，保护学生学习路径。
- 回答可以进入后续导出和复习材料，但必须脱敏。

## 8. 20 秒答辩话术

“ClassMate 的 Ask This Lesson 不是普通聊天入口。它只围绕当前课堂材料回答，并要求答案绑定证据。如果课堂里讲过，系统给出 grounded 答案和引用；如果只讲了一部分，就标注 partial；如果没有依据，就明确 not_found。这样学生不是得到一个看似流畅的答案，而是得到一个可追溯、可复习、可测验的学习结果。”
