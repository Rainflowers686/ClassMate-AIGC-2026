# OCR / PPT / 板书资料导入风险说明

## 1. OCR 错字可能导致知识点错误

风险：识别结果可能把 `0` 识别成 `O`，把 `1` 识别成 `l`，把公式符号识别错，进而影响知识点和题目。

缓解：

- 保留原始 OCR 文本作为证据来源，不把清洗结果伪装成原文。
- 证据 quote 定位失败时降级，不伪造证据。
- 后续引入课程术语表和公式保护规则。

答辩解释：

“OCR 错字是视觉识别的真实风险。ClassMate 的处理原则是保留来源、可追溯、可降级，而不是把不确定内容强行变成确定结论。”

## 2. 图片质量影响识别

风险：低清晰度、反光、倾斜、截断、手抖都会降低 OCR 质量。

缓解：

- UI 提醒用户尽量提供清晰截图或照片。
- 当前阶段允许手动修正 OCR 文本。
- 后续可增加图片质量提示，但不在复赛阶段夸大。

## 3. 公式识别困难

风险：高数、物理和 C++ 代码片段中的公式、上下标、希腊字母和符号容易识别错误。

缓解：

- 公式作为证据时优先使用短 quote。
- 对公式密集页面，允许用户手动编辑识别文本。
- 术语表可帮助识别常见符号和词汇，但不能替代真实公式 OCR。

## 4. PPT 页顺序可能错乱

风险：用户一次选择多张课件图时，系统可能无法判断真实讲课顺序。

缓解：

- 第一版按用户添加顺序排序。
- UI 显示资料序号和来源。
- 后续再做手动排序或自动页码识别。

## 5. 板书结构不规则

风险：板书可能包含箭头、圈注、擦除痕迹和非线性布局，直接转成纯文本会丢失结构。

缓解：

- 第一版把板书 OCR 当作补充资料，不要求完整还原布局。
- Evidence 只引用可定位短句或公式片段。
- 对结构不清楚的内容可降级为 partial。

## 6. 如果没有官方 OCR 文档，不能猜接口

风险：在没有官方接口文档、参数、鉴权和错误格式的情况下猜测真实 OCR 调用，会造成工程和合规风险。

缓解：

- 只保留 provider 抽象和接口预留。
- 等官方文档、截图或链接确认后再实现真实调用。
- 不写入真实访问凭据，不在文档中固化未确认接口。

## 7. 用户上传材料的隐私问题

风险：课件、板书照片、讲义截图可能包含姓名、学号、班级、教室或其他隐私信息。

缓解：

- 当前阶段不上传文件，只支持手动粘贴或本地元数据记录。
- 真机演示不拍摄含个人信息的材料。
- 导出报告只保留学习内容和资料来源摘要。

## 8. 不做第三方平台抓取的原因

风险：抓取网络课程视频、字幕或平台内容可能涉及版权、账号隐私、平台协议和技术稳定性。

策略：

- 不抓取第三方平台内容。
- 只处理用户本地文件或用户自行提供且有权使用的文本/字幕。
- 视频推荐只做白名单搜索入口，不下载、不解析、不后台抓取。

## 9. 如何解释“手动 OCR + 接口预留”

建议话术：

“当前版本不声称真实 OCR 已完成。我们先把课件、板书、讲义这些视觉资料纳入同一节课的资料包，并让 Timeline、Evidence、Ask This Lesson 和 Export 都能识别资料来源。后续接入官方 OCR 时，替换的是识别入口；证据链、校验和学习闭环不需要推翻。”

## 10. 对评委的边界表达

可以说：

- 已经准备课件 OCR、板书 OCR、讲义 OCR 的产品入口和测试素材。
- 当前可以通过手动 OCR 文本验证资料融合、证据引用和导出摘要。
- 后续优先接入官方 OCR 能力。

不要说：

- 真实 OCR 已经完成。
- 可以自动解析所有 PPT、PDF、图片和板书。
- 可以抓取网络平台视频或字幕。
- OCR 结果一定准确。
