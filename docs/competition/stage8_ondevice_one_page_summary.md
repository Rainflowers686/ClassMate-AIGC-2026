## ClassMate 智能课堂助手 — 模型分层架构

ClassMate 采用 **Cloud + OnDevice + LocalRule 三层模型架构**，在不同的网络和设备条件下提供最佳体验。

```
┌─────────────────────────────────────────┐
│              用户请求                      │
└─────────────────┬───────────────────────┘
                  ▼
    ┌─────────────────────────┐
    │  ① 端侧 BlueLM 3B (优先)  │  ← 本次新增
    │  本地推理 · 低延迟 · 隐私优  │
    └─────────┬───────────────┘
              │ 不可用 ↓
    ┌─────────▼───────────────┐
    │  ② 云端 Official BlueLM  │  ← 主力模型
    │  远程推理 · 能力最强       │
    └─────────┬───────────────┘
              │ 不可用 ↓
    ┌─────────▼───────────────┐
    │  ③ 本地规则引擎 (兜底)     │
    │  预设问答 · 保证基本可用    │
    └─────────────────────────┘
```

## 1. Official BlueLM 云端主路径

- **模型**：蓝心大模型（云端版）
- **定位**：主力推理引擎，处理复杂教学场景
- **场景**：Ask This Lesson（课件问答）、StudyReport（学习报告生成）、Practice（自适应练习）
- **安全**：`enable_thinking=false` guard 确保结构化 JSON 输出稳定；prompt 不记录不导出

---

## 2. 端侧 BlueLM 3B — 本次核心亮点

### 做了什么

- 真实接入官方 BlueLM 3B 端侧 SDK（通过 reflection bridge 对接 `LlmManager` / `TokenCallback` / `callVit`）
- Settings 诊断页面实时展示 SDK 状态（SDK_PRESENT / init 返回码 / 流式输出 / 错误码）
- 纯文本端侧推理可用，多模态 VIT 编码实验性对接

### 为什么做

| 价值维度 | 说明 |
|----------|------|
| **隐私保护** | 端侧对话文本和图片不离开设备。教育场景中未成年学生数据不出设备，合规性更强。 |
| **离线可用** | 教室、图书馆、自习室等网络不稳定环境中，端侧模型仍可提供基础答疑。 |
| **低延迟** | 本地推理无网络往返延迟，简单问答即时响应。 |
| **成本可控** | 端侧推理不消耗云端 API 配额，适合大规模部署。 |

### 诚实边界

- 3B 参数模型，回答质量**低于**云端 BlueLM —— 这是工程设计上的取舍
- 端侧模型定位为**补充**，不是**替代**云端
- 多模态为**实验性功能**，仅在 Settings 诊断页面可用，未接入学习主链路
- 真机兼容性受限于设备 SoC 和官方 SDK 白名单

## 3. 学习闭环差异化

ClassMate 不是通用 AI 聊天工具。我们的差异化在于**学习闭环**：

```
课堂录音 → ASR 转写 → 资料融合 → Ask 问答
    │                                    │
    └── 错题本 ← 自适应练习 ←── StudyReport ←┘
```

- **端侧模型的角色**：在弱网/隐私场景下为 Ask（简单问答）、Practice（选择题生成）提供本地推理
- **不与云端竞争**：复杂分析、长报告、深度推理始终走云端 BlueLM
- **Provider Chain**：无感切换，用户不需要关心"当前用的是哪个模型"

---

## 4. 隐私与弱网价值

### 隐私设计

- 端侧对话：文本 + 图片**全程不离开设备**
- 原始 prompt：**不保存、不导出**
- 模型输出：经过 redaction pipeline 脱敏
- 权限最小化：仅存储权限（模型文件）+ 网络权限（模型下载），无定位/通讯录/短信
- 密钥不入仓：`config.local.json` 在 `.gitignore` 中

### 弱网/离线场景

- 大学课堂、自习室、地铁通勤 —— 网络不稳定的常态场景
- 端侧模型提供基础答疑，不依赖网络
- 断网时三层兜底自动生效：端侧不可用 → 本地规则仍可响应
