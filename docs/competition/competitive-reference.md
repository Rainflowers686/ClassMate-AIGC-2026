# 听脑类产品功能参照与 ClassMate 差异化

本文只做产品能力维度参照，不照搬、不抄袭、不贬低竞品。这里把相关产品抽象为“录音转写/AI 总结类产品”。

## 1. 录音转写/AI 总结类产品的常见能力

这类产品通常围绕“把语音或音视频材料变成可读内容”展开，常见能力包括：

- 实时录音转写：课堂、会议、访谈中实时生成文字。
- 音视频上传：上传录音、视频或会议文件后生成文本。
- AI 总结：把长文本整理成摘要、重点、待办或纪要。
- 随时问答：基于记录内容提问，快速定位信息。
- 分享导出：把转写、总结和纪要导出或分享。
- 多场景适配：会议、课堂、访谈、播客、直播等。
- 词库/术语优化：对专业词、姓名、行业术语做识别优化。

这些能力很有参考价值，尤其是输入入口、录屏演示方式、导出体验和多场景组织。

## 2. ClassMate 的差异化

ClassMate 不把“记录下来”作为终点，而是把“学会”作为目标。

### 课堂知识点结构化

ClassMate 的输出是知识点时间线，而不是单段总结。知识点包含标题、摘要、重要性、难度和来源段落，更适合学习和复习。

### 原文证据链

每个关键结论尽量绑定课堂原文证据。用户和评委可以回到原文复核，而不是只能相信生成结果。

### 微测

ClassMate 自动生成面向学习的微测，覆盖理解、判断、应用、错因分析和迁移，帮助学生检查是否真的掌握。

### 自适应复习计划

系统根据知识点重要性、难度、答题结果和反馈生成 ReviewTask。复习不是固定列表，而是随学习行为变化。

### LearningStore 反馈闭环

答错、已掌握、太难、需要例题、证据不对等反馈会进入 LearningStore，进一步影响 Review 和 Weakness Hub。

### 官方 vivo BlueLM 合规路径

ClassMate 以 Official BlueLM 作为参赛合规主路径。Compatible demo 只是展示增强，LocalFallback 是兜底。

### 端侧/隐私兜底路线

未来端侧 3B 模型可以强化离线隐私模式，减少弱网和隐私场景下的依赖。

## 3. 借鉴但不照搬的功能路线

### P0 已有：课堂文本分析、证据链、复习闭环

当前优先证明 ClassMate 的核心差异：文本进入后可生成知识点、证据、微测、ReviewTask、LearningStore 和 Export。

### P1：Live 手动/模拟课堂

先把课堂伴学的产品状态闭环做出来：开始、暂停、继续、结束、片段、生成时间线。当前明确是手动/模拟，不伪装真实 ASR。

### P2：ASR/OCR 接入

后续接入 vivo ASR，把真实课堂转写接入 TranscriptSession；接入 vivo OCR，把课件、板书和教材页转成课堂文本。

### P3：端侧模型/向量聚合

端侧模型用于离线隐私兜底；文本向量用于跨课程知识点去重、相似课程聚合和复习任务合并。

## 4. 答辩表述建议

可以说：

> 我们参考了录音转写/AI 总结类产品的输入和导出体验，但 ClassMate 的差异化在学习闭环。我们不是只把课堂记录下来，而是把课堂内容转成可验证知识点、微测和自适应复习任务。

不要说：

- 不要说已经完成真实录音转写。
- 不要说已经完成 OCR。
- 不要说视频链接会自动抓取平台内容。
- 不要把 Compatible demo 说成官方合规主路径。
