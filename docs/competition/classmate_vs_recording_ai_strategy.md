# ClassMate 与录音转写类 AI 产品的追平和超越策略

## 定位

ClassMate 不是放弃录音转写类产品的基础能力，而是把这些能力作为课堂学习闭环的输入层。录音、转写、OCR、课件导入和随时问答是下一阶段必须追平的基础能力；ClassMate 的差异化在于把课堂材料转化为可验证知识点、微测、自适应复习、薄弱点和课程库。

换句话说：听课材料进入 ClassMate 后，不应只停留在“摘要”，而要进入“证据可追溯、问题可练习、复习可调度、反馈可更新”的学习系统。

## 必须追平的基础能力

### 实时转写

目标是让 Live Companion 从手动/模拟片段升级为真实课堂转写。第一阶段只做句段级文本与时间范围，不伪造词级同步，不声称声纹识别已经完成。

### 本地音频上传

支持用户选择自己的课堂录音文件，交给后续 ASR provider 转成 TranscriptSession。只处理用户本地文件，不抓取第三方平台内容。

### 本地视频上传

支持用户选择自己的课堂视频文件。第一阶段可以只抽取或手动提供字幕/转写文本，后续再接 ASR/OCR。明确不做网络视频爬取，不处理未授权内容。

### 字音同步

P1 做句段级同步：一句或一段对应音频时间范围。P2 在 ASR provider 返回词级时间戳时再做词级同步。没有词级时间戳时不可伪造。

### AI 随时问

Ask This Lesson 必须基于当前课程的知识点摘要和证据片段回答。没有证据时标注 partial 或 not_found，不做无根据扩写。

### 课件/图片/OCR

支持 PPT 截图、板书照片、PDF/讲义截图进入 OCR provider。第一阶段可以手动粘贴 OCR 文本或模拟 provider，后续优先接官方 vivo OCR。

### 术语表/热词表

按课程维护术语表，服务于 ASR 热词、OCR 纠错、模型输入约束、Quiz 术语一致性和导出解释。

### 导出分享

导出学习报告、知识时间线、证据链、微测题、复习计划、薄弱点摘要和多资料来源。导出必须避免敏感字段和供应商原始内容。

## 必须超越的学习闭环能力

### 原文证据链

每个知识点和题目都应追溯到课堂原文、转写片段、PPT OCR、板书 OCR、手动笔记或导入文本。证据无法定位时，不能伪造。

### 微测

模型或本地规则生成的微测题必须绑定知识点和证据。题目不仅验证“是否看过摘要”，还要验证概念、公式、方法、易错点和应用判断。

### 自适应复习

ReviewTask 根据答题结果和反馈更新优先级。答错、太难、需要例题会提高复习优先级；已掌握会降低优先级或退出薄弱点。

### Weakness Hub

薄弱点中心聚合反复错误、太难、需要例题、证据不对等反馈，让用户知道接下来该看证据、复习、找例题还是请求人工确认。

### Course Library

课程库按课程聚合历史课堂、知识点、题目和复习任务。它让 ClassMate 从单次总结工具变成持续学习系统。

### LearningStore

LearningStore 记录学习状态、复习任务和反馈计数，但不保存密钥、完整模型请求或供应商原始返回。它是自适应闭环的核心。

### 多资料融合证据来源

LessonMaterialBundle 统一 transcript、slide_ocr、blackboard_ocr、manual_note、imported_text 等来源，让最终学习结果能说明“这条证据来自哪里”。

## 不做或暂缓

### 第三方平台视频爬取

不做网络视频抓取、解析或下载。视频相关能力只允许处理用户本地文件，或提供白名单搜索入口。

### 云同步和团队协作

暂缓云同步、多人协作和团队空间，除非后续有服务器、隐私协议、账号体系和合规设计。复赛阶段优先保证本地学习闭环稳定。

### 声纹识别和底层音频增强

暂缓声纹识别、说话人分离、去噪增强等底层音频算法。第一阶段 speaker label 只做 teacher / student / unknown，并依赖服务侧或用户选择。

## 复赛策略

### P0：先修当前 bug

先保证现有官方 BlueLM 主链路、Live -> Timeline、History、Review、Export、UI 中文化和溢出问题稳定。生产链路不稳定时，不追加复杂入口。

### P1：展示 ASR/OCR/PPT 接入缝

用架构、模拟 provider、手动输入和诚实占位展示下一阶段可接 vivo ASR/OCR 的路径。重点是 LessonMaterialBundle 和 MaterialEvidenceRef，不夸大“已完成真实识别”。

### P2：展示学习闭环优势

演示从课堂材料到 Timeline、Evidence、Quiz、ReviewTask、Weakness Hub、Course Library、Export 的闭环。评委应看到 ClassMate 不是普通摘要工具，而是围绕学习行为持续优化。

## 答辩表达边界

- 可以说：ClassMate 已完成课堂文本分析、证据链、微测、复习任务、历史课程库、导出和手动 Live 课堂闭环。
- 可以说：ASR/OCR/PPT/端侧模型是下一阶段接入路线，已经规划数据模型和 provider 抽象。
- 不要说：已经完成真实 ASR、真实 OCR、真实视频解析、声纹识别、云同步或网络视频解析。
- 不要展示或讨论任何真实密钥。
