# Stage 7 Judge Q&A 50

## 1. 项目定位

### Q1：ClassMate 到底解决什么问题？
- 推荐回答：它把课堂资料转成可验证知识点、证据链、微测、复习任务和导出报告，目标是帮助学生课后真正复盘。
- 不该说：我们只是做一个录音总结工具。
- Proof：Home、Timeline、Review、Export Center。

### Q2：为什么不是普通 AI 摘要？
- 推荐回答：普通摘要难以追溯。ClassMate 要求知识点绑定课堂原文或资料来源，后续微测和复习也基于这些证据。
- 不该说：模型回答一定不会错。
- Proof：Evidence 详情页。

### Q3：复赛版最完整的闭环是什么？
- 推荐回答：导入课堂资料，生成知识时间线，查看证据，做微测，产生复习任务，标记需要多练，再导出报告。
- 不该说：所有后续能力都已经完成。
- Proof：完整录屏。

### Q4：目标用户是谁？
- 推荐回答：主要是需要课后复盘的学生，也适合教师或助教整理课堂材料。
- 不该说：它已经覆盖所有教育场景。
- Proof：课程库和学习报告。

### Q5：创新点是什么？
- 推荐回答：官方模型路径、证据可追溯、微测、自适应复习和练习闭环的组合。
- 不该说：只是换皮调用大模型。
- Proof：Provider 设置、Evidence、Review。

## 2. 与录音转写类产品差异

### Q6：和录音转写产品最大区别是什么？
- 推荐回答：录音转写产品强在记录和总结；ClassMate 重点是把资料变成学习任务和复习计划。
- 不该说：竞品没有价值。
- Proof：Review、需要多练、Course Library。

### Q7：你们追平了哪些输入能力？
- 推荐回答：文本、Markdown、字幕/转写稿、手动 OCR 资料流、Live 手动和系统 ASR 实验模式。
- 不该说：真实 vivo ASR/OCR provider 已经完成。
- Proof：Import Flow、Live。

### Q8：为什么不直接做实时录音产品？
- 推荐回答：复赛先完成学习闭环，实时 ASR 作为实验模式和后续 provider 接入方向，避免为了录音牺牲证据和复习质量。
- 不该说：录音能力不重要。
- Proof：Capability Roadmap。

### Q9：你们是否支持问答？
- 推荐回答：支持 Ask This Lesson，限定在本节课资料中找依据，分 grounded、partial、not_found。
- 不该说：它是任意通用问答助手。
- Proof：Ask 页面。

### Q10：你们比录音总结类产品强在哪里？
- 推荐回答：更强的是学习闭环：知识点、证据、微测、复习、练习、报告。
- 不该说：我们全面超过所有竞品。
- Proof：Timeline + Quiz + Review。

### Q11：为什么强调证据链？
- 推荐回答：学生需要知道知识点来自课堂哪一句，评委也能验证模型没有随意编造。
- 不该说：证据链能保证 100% 正确。
- Proof：Evidence quote。

### Q12：你们做分享导出了吗？
- 推荐回答：有 Export Center，支持多格式和系统分享。
- 不该说：生成真正原生 Office 文件。
- Proof：Export Center。

### Q13：为什么不做第三方平台视频解析？
- 推荐回答：涉及版权和平台规则风险，复赛版只做合规搜索入口或用户自有资料导入。
- 不该说：我们会抓取平台内容。
- Proof：Practice Search 文案。

## 3. BlueLM 合规与模型路径

### Q14：官方 BlueLM 在系统里是什么位置？
- 推荐回答：Official BlueLM/qwen3.5-plus 是参赛合规主路径，用于课堂分析和相关智能能力。
- 不该说：Compatible Demo 是主路径。
- Proof：Settings Provider。

### Q15：Compatible Demo 是什么？
- 推荐回答：它是展示增强路径，用来演示 OpenAI-compatible 兼容能力，不作为参赛官方主路径。
- 不该说：它替代官方路径。
- Proof：Settings Provider。

### Q16：LocalFallback 为什么保留？
- 推荐回答：网络、权限或模型失败时保证基本演示和学习流程不中断，同时清楚标注 fallback。
- 不该说：本地兜底等同于官方模型质量。
- Proof：结果来源标签。

### Q17：qwen3.5-plus 为什么关闭 thinking？
- 推荐回答：课堂分析需要稳定、简洁、可解析输出，关闭深度思考可减少长推理内容和超时风险。
- 不该说：thinking 没有价值。
- Proof：协议测试或代码 guard 检查结果。

### Q18：密钥怎么保护？
- 推荐回答：只走本机配置或 debug 导入，不进仓库、不进日志、不进导出、不进截图。
- 不该说：密钥可以写在 README 里。
- Proof：Secrets Scan。

### Q19：模型失败怎么办？
- 推荐回答：会标注失败原因，并按 profile 规则 fallback，本地兜底不会伪装成 BlueLM 成功。
- 不该说：永远不会失败。
- Proof：日志短状态。

## 4. 证据链与防胡编

### Q20：如何防止模型胡编知识点？
- 推荐回答：要求 evidence quote，并由本地 resolver/validator 检查证据可定位，失败项不进入最终结果或降级。
- 不该说：完全靠模型自觉。
- Proof：Evidence Validator 测试。

### Q21：Ask 如何防胡编？
- 推荐回答：Ask 回答必须关联本节课证据，无法定位时显示 partial 或 not_found。
- 不该说：它可以回答任何问题。
- Proof：Ask 三类问题。

### Q22：证据来自哪里？
- 推荐回答：来自导入文本、字幕/转写稿、手动 OCR、Live 片段等 MaterialBundle 来源。
- 不该说：证据可以由模型生成。
- Proof：来源 marker。

### Q23：证据 quote 被模型改写怎么办？
- 推荐回答：本地定位失败就不能当作强证据，会降级或进入 fallback。
- 不该说：改写也算证据。
- Proof：解析/校验日志。

### Q24：微测题怎么保证有依据？
- 推荐回答：题目解释和 evidence quote 需要能回到课堂材料，不能脱离本课资料。
- 不该说：题目越多越好，不用校验。
- Proof：Quiz 解释和证据。

### Q25：如果课堂文本本身很短呢？
- 推荐回答：系统会生成较少知识点或使用本地兜底，避免强行编造。
- 不该说：短文本也一定生成完整大纲。
- Proof：短文本测试。

## 5. ASR/OCR/多模态边界

### Q26：真实 vivo ASR 接了吗？
- 推荐回答：复赛版有系统 ASR 实验模式和接口规划，vivo ASR provider 是后续接入项。
- 不该说：真实 vivo ASR provider 已经接入。
- Proof：Settings 路线图。

### Q27：真实 vivo OCR 接了吗？
- 推荐回答：当前是手动 OCR 资料流，后续接 vivo OCR provider。
- 不该说：图片已经自动识别。
- Proof：OCR 导入文案。

### Q28：Live ASR 是否保存音频？
- 推荐回答：不保存原始音频，不后台录音，只依赖设备系统识别结果。
- 不该说：会保存完整课堂录音。
- Proof：Live ASR 说明。

### Q29：支持自动说话人分段吗？
- 推荐回答：当前支持手动说话人标签，自动分段待接入 provider。
- 不该说：声纹识别已完成。
- Proof：Transcript Editor。

### Q30：为什么不做声纹身份识别？
- 推荐回答：隐私和算法复杂度高，复赛优先学习闭环，后续也要谨慎合规。
- 不该说：已经能识别具体人。
- Proof：Roadmap 暂缓项。

### Q31：底噪处理怎么做？
- 推荐回答：先依赖系统或 provider 能力，不自研底层音频算法。
- 不该说：已实现自研降噪。
- Proof：Capability Roadmap。

### Q32：视频文件怎么处理？
- 推荐回答：优先支持用户提供的字幕/转写稿或本地文件元数据，不抓取第三方平台。
- 不该说：能自动下载平台视频。
- Proof：Import Hub。

### Q33：多来源会不会混乱？
- 推荐回答：MaterialBundle 统一来源，证据 marker 区分 transcript、OCR、manual note 等。
- 不该说：来源不重要。
- Proof：Timeline 来源 marker。

## 6. 隐私安全

### Q34：会上传学生隐私吗？
- 推荐回答：只有用户主动调用模型时才发送课堂文本；导出和历史不包含密钥或内部请求详情。
- 不该说：任何数据都不会出设备。
- Proof：Settings 隐私说明。

### Q35：导出会泄漏敏感信息吗？
- 推荐回答：导出只包含学习报告内容，不包含密钥、内部请求、原始供应商响应或模型内部推理。
- 不该说：无需检查导出。
- Proof：导出文件检查。

### Q36：日志里有什么？
- 推荐回答：只有短状态、provider、latency、错误标签等脱敏信息。
- 不该说：日志里保留完整模型交互。
- Proof：Logs 卡。

### Q37：GitHub 是否追踪本地配置？
- 推荐回答：不追踪，本地配置和构建产物都在安全检查清单中。
- 不该说：配置文件可以提交。
- Proof：git ls-files 检查。

### Q38：如果密钥曾经暴露怎么办？
- 推荐回答：立刻重置密钥，重新跑 secrets scan 和历史检查。
- 不该说：暴露一次没关系。
- Proof：Security checklist。

### Q39：截图时要注意什么？
- 推荐回答：不拍密钥、debug 输入框、本地路径、私密账号和鉴权内容。
- 不该说：所有设置页都可以无脑录屏。
- Proof：Proof screenshot list。

### Q40：是否有权限风险？
- 推荐回答：按功能最小化权限；录音实验模式需要明确用户操作和权限说明。
- 不该说：后台静默录音。
- Proof：Manifest 权限检查。

## 7. 学习闭环与复习算法

### Q41：复习任务从哪里来？
- 推荐回答：来自校验通过的知识点和用户反馈，任务有来源课程和原因。
- 不该说：凭空生成任务。
- Proof：Review 任务卡。

### Q42：需要多练如何触发？
- 推荐回答：错题、太难、用户标记等信号会进入学习状态，形成练习建议。
- 不该说：只是 UI 标签。
- Proof：Review/Practice Search。

### Q43：错题本有什么价值？
- 推荐回答：它把答错记录和知识点绑定，帮助学生回到证据和练习。
- 不该说：只是保存错题文字。
- Proof：Adaptive Practice 若完成则展示，否则说明待验证。

### Q44：学习状态保存什么？
- 推荐回答：保存复习任务、答题结果和反馈计数，不保存密钥、内部请求或模型原始响应。
- 不该说：保存所有原始模型数据。
- Proof：LearningStore 说明。

### Q45：复习算法复杂吗？
- 推荐回答：复赛版是可解释规则优先，答错、太难、需要多练会提高优先级，已掌握会降低优先级。
- 不该说：已经是完整商业级推荐系统。
- Proof：Review 反馈变化。

### Q46：如何避免学生盲目刷题？
- 推荐回答：练习入口基于本课知识点和证据，报告里保留搜索词，用户自己判断资源质量。
- 不该说：平台资源由我们抓取审核。
- Proof：Practice Search 文案。

## 8. 商业化/决赛路线

### Q47：决赛最值得补什么？
- 推荐回答：真实 vivo ASR/OCR provider、多来源资料融合、更完整学习画像和演示视频。
- 不该说：复赛已完成全部能力。
- Proof：Roadmap。

### Q48：商业化方向是什么？
- 推荐回答：个人学习助手、课堂复盘、教师助教材料整理，以及隐私优先的本地兜底模式。
- 不该说：立即做云同步变现。
- Proof：Course Library + Export。

### Q49：最大风险是什么？
- 推荐回答：长文本模型稳定性、证据定位召回、ASR/OCR provider 接入和移动端产品完成度。
- 不该说：没有风险。
- Proof：Risk notes。

### Q50：为什么这个项目适合继续做？
- 推荐回答：已经有官方模型主路径和闭环骨架，后续接入 ASR/OCR 会自然增强输入，而不是推翻架构。
- 不该说：只靠换模型就能完成。
- Proof：MaterialBundle + Timeline + Review。
