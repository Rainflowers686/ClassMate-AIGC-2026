# 评委可能提问与回答

要求：回答诚实，不夸大；强调官方 BlueLM 合规主路径、证据链和复习闭环；未完成的 ASR/OCR 明确说未完成。

## 1. 你们和听脑/录音总结类产品有什么区别？

听脑类产品通常偏记录、转写和总结。ClassMate 更偏学习闭环：课堂内容会被转成知识点时间线，每个结论尽量绑定原文证据，然后生成微测和复习任务，用户答题和反馈还会继续优化学习状态。

## 2. 为什么不是简单调用大模型总结？

普通总结无法保证学生真的理解，也难以复核结论来源。ClassMate 要求模型输出结构化学习草稿，再由本地证据解析和校验组装成知识点、微测、复习任务。重点不是“总结得像”，而是“能验证、能练习、能复习”。

## 3. 证据链怎么保证可靠？

模型返回的证据短句必须能在课堂原文中定位。系统会用 EvidenceResolver 定位，再由 EvidenceValidator 检查 span 是否真实指向原文。定位失败的内容不能伪造成证据。

## 4. 微测题是怎么生成和校验的？

微测题来自分析结果中的知识点，题型服务概念理解、判断、应用、错因分析和迁移。题目必须关联知识点，并带可定位证据。校验器会检查题目引用的知识点是否存在、是否有正确选项、是否有证据。

## 5. 复习计划怎么自适应？

LearningStore 会记录答题结果和反馈事件，例如答错、已掌握、太难、需要例题、证据不对。ReviewTask 根据这些信号调整优先级、状态和复习建议，Weakness Hub 也从这些计数中聚合薄弱点。

## 6. 如果模型输出错了怎么办？

首先由 JSON 解析、证据解析和 ResultValidator/EvidenceValidator 拦截不合格结果。模型失败、解析失败或校验失败都会进入 fallback 路径。用户还可以标记“证据不对”或“不准确”，进入复核和复习反馈链路。

## 7. 为什么要保留 LocalFallback？

LocalFallback 是体验保底，不是主智能能力。它保证没有网络、没有凭据或模型失败时 App 仍可展示可用结果，同时会清楚标注来源，避免把兜底结果伪装成官方模型输出。

## 8. Compatible/DeepSeek demo mode 是否影响比赛合规？

不影响。Compatible demo mode 只是展示增强，用于说明系统具备兼容能力。参赛合规主路径是 Official BlueLM，文档和 UI 都会区分这两者。

## 9. 官方 BlueLM 在系统中是什么位置？

Official BlueLM 是主 provider。正常配置后，课堂文本分析优先走 BlueLM/qwen3.5-plus，失败时再进入后续 fallback。Settings 中也会展示当前 provider order 和诊断结果。

## 10. qwen3.5-plus 为什么要关闭 thinking？

qwen3.5-plus 默认可能产生较长的思考内容。课堂分析需要稳定、紧凑的 JSON 学习草稿；关闭 thinking 可以减少超时风险，避免无关思考内容影响长课堂输出稳定性。

## 11. 为什么现在 Live Companion 还是手动/模拟转写？

本阶段不做真实录音和 ASR，是为了避免权限、隐私和实现不稳定风险。当前先完成课堂伴学状态闭环：标题、开始/暂停/继续/结束、片段、生成时间线。后续接入 ASR 时，真实转写片段可以直接进入这个结构。

## 12. ASR/OCR 后续怎么接？

ASR 可以把课堂语音转成 TranscriptSegment，进入 Live Companion；OCR 可以把课件、板书或教材页转成课堂文本，进入 Import Hub。两者接入后仍复用同一套分析、证据、微测和复习链路。

## 13. 是否会上传学生隐私数据？

当前文本分析在官方 BlueLM 路径下需要把用户提供的课堂文本发给模型服务。产品需要提醒用户只导入允许处理的内容。密钥、日志、导出和历史数据不会保存认证信息或厂商原始返回。

## 14. 密钥怎么保护？

真实 AppID/AppKEY/API key 不写入仓库，不写入 README 或 proof 材料。Debug 导入只在本机调试使用，UI 只显示 masked 状态。提交前跑 secrets scan，并检查 Git 不追踪本地配置和构建产物。

## 15. 导出文件会不会泄漏请求上下文或模型响应？

不会按设计导出这些内容。导出只包含学习业务数据，例如知识点、证据、微测、复习任务、课程库、薄弱点和 MindMap。发布前仍会人工搜索敏感词并跑安全检查。

## 16. 为什么不做云同步/团队协作？

当前比赛阶段优先证明课堂理解和学习闭环。云同步和团队协作会引入账号、权限、隐私和后端复杂度，不是当前最关键的差异化能力。

## 17. 端侧 3B 模型有什么价值？

端侧模型适合离线兜底、隐私模式和弱网场景。它可以作为 LocalFallback 的增强版本，在不上传课堂内容的情况下提供更强的本地理解能力，是决赛阶段很有价值的扩展方向。

## 18. 如果网络不好怎么办？

Official BlueLM 失败时，系统会记录安全短日志，并进入 fallback。用户仍能得到本地兜底结果，Review 和 History 不会因为网络失败完全中断。

## 19. 课程库和 History 的关系？

History 是每次课堂分析记录；Course Library 是在 History 之上的课程级聚合。它按课程名聚合多次课堂，展示课堂次数、知识点总数、微测题总数和待复习任务数。

## 20. LearningStore 存什么，不存什么？

LearningStore 存复习任务、答题尝试、反馈计数和学习状态这类业务数据。它不存真实密钥、不存认证头、不存完整请求上下文、不存厂商原始返回。

## 21. Weakness Hub 怎么判断薄弱点？

它基于 ReviewTask 和反馈计数聚合：反复答错、太难、需要例题、证据不对或需要人工复核都会进入薄弱点。已掌握会降低优先级或移出薄弱点。

## 22. 视频推荐是否会爬取平台内容？

不会。视频推荐只生成白名单来源的搜索入口，不爬取、不下载、不解析平台内容，也不替代课堂原文证据。

## 23. 你们的创新点是什么？

创新点是把大模型输出约束为可验证学习结构：知识点要有证据、题目要绑定知识点和证据、复习任务要根据答题和反馈更新。它不是一次性总结，而是持续学习闭环。

## 24. 决赛前最值得补的能力是什么？

优先补真实 ASR/OCR 接入和 proof 材料。ASR 能让 Live Companion 从手动演示变成真实课堂伴学；OCR 能让课件和板书进入同一分析链路。

## 25. 目前最大风险是什么？

最大风险是长课堂文本的模型输出稳定性和真机网络环境。我们通过紧凑输出协议、证据校验、fallback 和安全诊断来降低风险，但仍需要真实样例持续验证。

## 26. 为什么要做 Capability Roadmap？

它让评委看到我们不是临时堆功能，而是沿着 vivo AIGC 能力规划产品路线。同时它清楚区分已接入、规划中和展示增强，避免夸大。

## 27. Ask This Lesson 怎么保证不乱答？

它应该基于当前课程的知识点和证据作答。如果资料中没有明确依据，就标为 partial 或 not_found，而不是编造证据。后续接模型问答时也会要求结构化 JSON 和证据定位。

## 28. 为什么导入音频、视频、OCR 现在只是占位？

这些能力涉及权限、媒体解析、模型接口和隐私风险。当前阶段先把学习闭环做稳，并诚实展示占位。决赛阶段再逐个接入官方能力。

## 29. 你们如何证明没有泄漏密钥？

通过三层证明：Git 不追踪本地配置和构建产物；secrets scan 通过；proof 截图、导出文件、日志和文档都只展示 masked 或脱敏内容。

## 30. 如果模型给出没有证据的知识点，你们会保留吗？

不会作为可信结果直接保留。没有可定位证据的知识点会被剔除、降级或导致校验失败，最终进入 fallback 或提示需要复核。
