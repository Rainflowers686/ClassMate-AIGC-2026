# 听脑类录音转写/AI 总结产品功能差距与超越路线

本文只从产品能力维度抽象比较，不照搬、不抄袭、不贬低竞品。这里统一称为“听脑类录音转写/AI 总结产品”。

## 1. 输入采集能力差异

听脑类产品通常强在输入采集：

- 实时录音转写。
- 本地音频/视频上传。
- 多场景记录，例如会议、课堂、访谈。
- 可能支持术语词库来提升转写准确度。

ClassMate 当前强在文本进入后的学习加工：

- 已支持粘贴文本、txt/md。
- 已有 Live manual 课堂伴学骨架。
- 音频、视频、OCR、ASR 仍是下一阶段接入准备。

追平方向：

- ASR 接入 Live Companion。
- OCR 接入课件、板书、讲义截图。
- 本地音频/视频只处理用户授权材料。

不做：

- 不做第三方平台网络视频爬取。
- 不抓取未授权内容。

## 2. 转写与记录层差异

听脑类产品核心通常是“记录准确、转写可读、可回放”。

ClassMate 当前记录层仍较轻：

- Live Companion 是手动/模拟转写。
- 还没有真实时间轴回放。
- 没有多人 speaker 识别。

下一阶段需要补：

- `TranscriptSession` 标准化。
- 句段级时间戳。
- teacher/student/unknown speaker 标签。
- 术语热词提升 ASR 质量。

## 3. AI 总结与问答差异

听脑类产品通常提供总结、章节、待办和基于记录的问答。

ClassMate 的差异在于：

- 不是只输出总结，而是输出知识点时间线。
- 每个关键结论绑定证据。
- Ask This Lesson 应基于本课知识点和证据，资料中没有依据时标为 not found 或 partial。

追平方向：

- Ask This Lesson 接入 provider，但必须保持证据约束。
- 支持基于 transcript、OCR 和手动笔记的融合问答。

## 4. 学习闭环差异

这是 ClassMate 的核心壁垒。

听脑类产品通常记录完成后停在总结、搜索和分享。ClassMate 继续进入：

- 微测。
- 答题反馈。
- ReviewTask。
- LearningStore。
- Weakness Hub。
- Course Library。
- Export 学习报告。

超越路线：

- 让每个知识点都有证据、微测和复习状态。
- 让用户反馈影响复习优先级。
- 让课程库聚合跨课堂学习进展。

## 5. 导出分享协作差异

听脑类产品通常在分享和协作上更成熟：

- 分享链接。
- 团队空间。
- 多设备同步。
- 文档导出。

ClassMate 当前：

- 已支持 md/html/txt 导出。
- 不做云同步和团队协作。

暂缓原因：

- 云同步需要服务器、账号体系、权限管理、隐私合规和成本预算。
- 当前比赛阶段优先证明学习闭环和官方模型路径。

## 6. 安全隐私差异

ClassMate 当前安全策略：

- 真实凭据不入仓。
- Debug 导入本地化。
- 日志和导出只保留学习业务数据。
- 不导出模型请求上下文、完整厂商原始返回或内部思考字段。
- Compatible demo 与 Official BlueLM 合规主路径明确区分。

下一阶段要加强：

- 本地材料导入前的隐私提示。
- 导出前敏感词检查。
- 端侧模型隐私兜底。
- 文本审核能力调研。

## 7. UI 产品成熟度差异

听脑类产品通常更成熟：

- 录音入口清晰。
- 转写列表、回放、问答、导出路径顺滑。
- 大量真实场景打磨。

ClassMate 当前正在补齐：

- Focus 默认学习体验。
- Flow Live 课堂体验。
- Course Library。
- Weakness Hub。
- Capability Roadmap。

下一步重点：

- 中文化和无乱码。
- 全屏 UI 一致性。
- 真机文字溢出检查。
- 演示视频和 proof 截图。

## 8. ClassMate 必须追平的功能

P0 / P1 追平项：

- 真实 ASR 转写入口。
- 本地音频上传转写。
- PPT/板书/讲义 OCR。
- 基于本课材料的问答。
- 更完整的导出报告。
- 术语表增强。
- 句段级字音同步。

## 9. ClassMate 应该超越的功能

核心超越点：

- 原文证据链。
- 微测。
- 自适应复习计划。
- LearningStore 反馈闭环。
- Course Library 跨课堂聚合。
- Official BlueLM 合规主路径。
- LocalFallback 和端侧隐私兜底路线。
- 资料融合证据链：文本、ASR、OCR、笔记都能回到来源。

## 10. 不做/暂缓的功能及原因

- 第三方平台网络视频爬取：版权、平台规则和合规风险高。
- 未授权字幕/音视频抓取：不符合安全和合规原则。
- 云同步/团队协作：需要服务器、账号、存储和隐私合规方案，当前暂缓。
- 声纹识别：没有明确能力前不声称完成。
- 词级字音同步：没有真实 ASR 时间戳前不伪造。

## 11. 路线总结

ClassMate 下一阶段要追平听脑类产品的基础输入能力：ASR、OCR、材料导入和问答；同时在学习闭环上超过它：证据链、微测、复习调度、Weakness Hub 和 Course Library。
