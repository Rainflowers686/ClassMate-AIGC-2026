# ClassMate Review Readiness 报告

本文用于比赛评委材料、答辩准备和 README 后续复用。内容只描述当前已实现能力和明确规划，不夸大未完成能力。

## 1. 项目定位

ClassMate 不是单纯的录音总结工具，也不是只把课堂内容压缩成摘要的笔记 App。

ClassMate 的定位是“课堂理解 + 原文证据链 + 微测 + 自适应复习 + 反馈闭环”的学习系统：

- 课堂理解：把课堂文本转为知识点时间线，而不是只给一段摘要。
- 原文证据链：知识点、微测和复习建议都要能回到原文片段。
- 微测：题目服务于理解、判断、应用、错因分析和迁移，而不是让用户记住原文句子。
- 自适应复习：LearningStore / ReviewTask 根据答题和反馈更新优先级。
- 反馈闭环：已掌握、太难、需要例题、证据不对等反馈会影响复习队列和薄弱点中心。

## 2. 当前已真实跑通的能力

### 官方与兜底路径

- vivo BlueLM 官方文本生成路径，主模型为 qwen3.5-plus。
- qwen3.5-plus 请求关闭深度思考，避免长课堂 JSON 输出被 reasoning 拖慢。
- Compatible demo mode，用于展示 OpenAI-compatible 兼容能力，不作为官方合规主路径。
- LocalFallback，用于无网络、无凭据或主路径失败时的本地兜底。

### 学习主流程

- 课堂文本分析。
- 知识点时间线。
- 原文证据链。
- 微测题目、答案、解释和证据。
- LearningStore / ReviewTask。
- History / Course Library。
- Markdown / HTML / TXT Export。
- MindMap。
- 视频白名单搜索推荐。
- Weakness Hub。
- Live Companion 手动课堂。

## 3. 诚实占位与后续接入

以下能力当前没有声称完成，只保留产品入口、数据结构或路线图：

- ASR：Live Companion 当前是手动/模拟转写演示，未接入真实语音识别。
- OCR：图片/OCR 导入当前是占位，未接入真实 OCR。
- 音频/视频导入解析：当前不解析媒体文件，不做后台转写。
- 网络视频链接：不抓取平台内容，只支持用户提供课堂文本/字幕；视频推荐只走白名单搜索 URL。
- 白噪音真实音频：Flow 方向只作为主题/体验方向，未实现真实音频播放。
- 端侧 3B 模型：作为决赛扩展路线，当前未接入。
- 文本向量/相似度：作为跨课程知识聚合路线，当前未接入。
- 文本审核：作为导出/分享前安全检查路线，当前未接入。

## 4. 和听脑类产品的差异

听脑类产品通常偏“记录、转写、总结”：

- 重点是把会议或课堂音频变成文本。
- 重点是总结、纪要、待办和检索。
- 学习后的微测、复习调度和证据校验通常不是核心闭环。

ClassMate 偏“学习闭环”：

- 输入可以来自课堂文本、手动课堂片段或未来 ASR/OCR。
- 输出不是单纯摘要，而是可追溯的知识点时间线。
- 每个关键结论都尽量绑定原文证据。
- 微测和复习任务用于提升掌握度。
- 用户反馈会回流到 LearningStore、ReviewTask 和 Weakness Hub。

## 5. 安全与合规

- 密钥不入仓：真实 AppID/AppKEY/API key 只允许在本机本地配置或 debug import 中使用。
- Debug 导入只服务本次会话或本机调试，不把明文展示给 UI。
- 日志脱敏：只记录 provider、状态、延迟、短错误码、fallback、校验状态等安全字段。
- 导出不含密钥、完整 prompt/messages、vendor response body 或 reasoning_content。
- HistoryStore 和 LearningStore 只存学习业务数据，不存凭据或原始厂商响应。
- Compatible 仅作为展示增强路径；Official BlueLM 是参赛合规主路径。

## 6. 决赛扩展路线

- ASR -> Live Companion 真转写：课堂录音或实时语音进入 transcript，再复用现有分析链路。
- OCR -> 课件/板书导入：课件、板书、教材页转文本，再复用证据链分析。
- 端侧 3B -> 离线隐私兜底：在弱网或隐私模式下提供更强本地能力。
- 文本向量 -> 跨课程知识聚合：相似课、知识点去重、复习任务合并。
- 文本审核 -> 分享/导出前安全检查：降低证明材料、导出文件和分享内容风险。

## 7. 答辩表达建议

可以这样概括：

> ClassMate 的核心不是“帮学生记笔记”，而是把课堂文本变成可验证、可练习、可复习的学习闭环。它使用 vivo BlueLM 生成学习草稿，本地校验证据并组装成知识点、微测和复习任务；模型失败时有 LocalFallback 保底，但不会把兜底包装成官方模型能力。未接入的 ASR/OCR/端侧能力都在路线图中诚实标注为规划中。

不要这样表达：

- 不要说已经完成真实 ASR。
- 不要说已经完成 OCR。
- 不要说已经完成端侧 3B 模型。
- 不要说 Compatible demo 是官方路径。
- 不要展示或描述任何真实密钥。
