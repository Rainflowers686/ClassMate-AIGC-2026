# ClassMate 复赛 5-7 分钟演示脚本

用途：现场讲解、录屏配音、答辩彩排。  
原则：专业、可直接念、不夸大；不展示真实密钥；未完成能力明确说“规划中”。

## 0:00-0:30 项目定位

大家好，我们的项目是 ClassMate，一个面向课堂学习的 AI 助手。

它不是普通的录音总结工具，也不是简单把一节课压缩成摘要。ClassMate 的核心链路是：

课堂内容进入系统后，先被结构化成可验证的知识点；每个知识点都尽量绑定原文证据；然后系统生成面向学习的微测题；用户答题和反馈会继续影响复习任务和薄弱点中心。

所以我们关注的是“课堂内容 -> 可验证知识点 -> 微测 -> 自适应复习 -> 反馈优化”的完整学习闭环。

## 0:30-1:30 官方 BlueLM 合规路径

现在先看 Settings。这里展示了三个路径：

第一，Official BlueLM。这是我们参赛的官方合规主路径，当前使用 vivo AIGC 文本生成能力，主模型是 qwen3.5-plus。

第二，Compatible Demo。它只是展示增强，用来说明系统可以兼容类似接口的演示环境，但它不是我们的官方参赛主路径。

第三，LocalFallback。它是本地兜底能力，保证没有网络、没有配置或模型调用失败时，App 仍然不会空白。

这里需要强调：真实凭据不会进入代码仓库，也不会展示在屏幕上。Debug 导入只显示 masked 状态，例如 credential_present=true，不展示明文。

接着我们点击 BlueLM 连接测试。成功时可以看到 provider=BLUELM、status=OK、HTTP 状态和短内容预览。失败时也只显示短诊断信息，不展示密钥、完整请求或模型原始返回。

## 1:30-2:30 导入课堂与知识时间线

接下来进入 Import Hub。

当前版本已经支持三种真实可用的输入：直接粘贴课堂文本、导入 txt 文件、导入 md 文件。音频、视频、OCR 和网络视频链接现在只是诚实占位，不会伪装成已经接入。

我们粘贴一段课堂文本，点击 Generate knowledge timeline。系统会按当前 provider profile 调用主链路；在 official_bluelm 模式下优先走 BlueLM/qwen3.5-plus。

生成完成后进入 Knowledge Timeline。这里不是普通摘要，而是按知识点组织的课堂理解结果。每个知识点有重要性、难度、摘要和证据入口。

点击一个知识点可以进入证据详情，看到它来自课堂原文的哪一段。这个 evidence quote 是 ClassMate 与普通总结工具的关键差异：它让模型结论可以被复核，而不是只能相信一段生成文本。

## 2:30-3:30 微测与复习闭环

现在进入 Quiz。

微测不是为了考“哪句话和原文最像”，而是围绕概念理解、判断、应用、错因分析和迁移来生成。用户答题后，可以看到正确或错误反馈、解释，以及对应证据。

答题结果会进入学习状态。接着进入 Review 页面。

这里的任务不是固定清单，而是根据知识点重要性、难度、答题情况和用户反馈生成。用户可以标记“已掌握”“太难”“需要例题”“证据不对”。这些反馈会影响任务优先级，也会进入 Weakness Hub。

Weakness Hub 展示的是当前真正需要关注的薄弱点，比如反复答错、觉得太难、需要例题或证据需要复核的知识点。

## 3:30-4:30 Live Companion

接下来展示 Live Companion。

这里我们非常明确：当前是手动/模拟转写演示，暂未接入真实 ASR。它不会申请录音权限，也不会伪装成已经完成实时语音识别。

我们输入课程标题，点击 Start。然后手动添加几条课堂片段。可以看到课堂计时、片段数量和最近片段。

暂停、继续、结束课堂后，系统会把这些片段拼接成正式课堂文本。点击 Generate timeline 后，它不会绕开主链路，而是复用同一套 BlueLM / Compatible / LocalFallback provider 体系，生成知识时间线、证据链、微测和复习任务。

这样 Live Companion 先形成了课堂伴学的产品骨架，后续接入 vivo 长语音转写或实时短语音识别时，可以直接把真实转写片段接入这个结构。

## 4:30-5:30 History / Course Library / Export

现在进入 History。

History 保存每次课堂分析记录。Course Library 会按课程聚合历史记录，显示课程名、最近学习时间、课堂次数、知识点总数、微测题总数和待复习任务数。

点击某个课程，可以重新打开具体课堂，继续查看知识点时间线和证据链。删除一条 History 记录不会清空 provider 配置，也不会直接删除 LearningStore 中的复习任务。

接下来演示 Export。ClassMate 可以导出 Markdown、HTML 和 TXT 学习报告。报告包含知识时间线、知识点与证据、微测、复习任务、课程库摘要、薄弱点摘要、MindMap 和白名单视频搜索入口。

导出内容不包含密钥、完整请求上下文、模型原始返回或模型思考字段。这一点是我们 proof 材料和安全检查的重要要求。

## 5:30-6:30 扩展路线

最后看 Capability Roadmap。

当前已经接入的是官方 BlueLM 文本生成主路径。规划中的能力包括：

vivo OCR，用于课件、板书和教材页导入；ASR 和长语音转写，用于 Live Companion 的真实转写；文本向量和相似度，用于跨课程知识点去重、相似课聚合和复习任务合并；文本审核，用于导出和分享前的安全检查；端侧 3B 模型，用于离线隐私兜底；Flow 学习陪伴方向，则可以扩展白噪音和沉浸课堂体验。

这些能力我们现在不会夸大成已经完成。当前完成的是学习闭环骨架和官方文本模型主链路，后续能力会沿着这个骨架继续接入。

## 20 秒收尾话术

总结一下，ClassMate 的创新点不在于“又做了一个课堂总结”，而在于把课堂内容变成可验证、可练习、可复习、可反馈优化的学习闭环。

我们用 vivo BlueLM 作为官方主路径，用证据链保证可复核，用微测和 LearningStore 让学习持续发生，用 LocalFallback 保证体验不中断。未完成的 ASR、OCR 和端侧能力也已经在产品结构中留好接口，适合决赛继续扩展。
