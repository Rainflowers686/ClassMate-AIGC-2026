# Stage 8A 端侧模型复赛答辩话术

## 1. 推荐说法

ClassMate 的主路径仍然是 Official BlueLM/qwen3.5-plus。端侧 BlueLM 3B 是下一阶段的本地智能兜底，用于弱网、隐私敏感和小任务增强，例如问这节课、学习报告建议、复习解释和练习提示。

## 2. 不要夸大的边界

- 不说端侧 SDK 已经全量接通，除非真机 smoke 已经通过。
- 不说端侧模型替代云端主路径。
- 不说本地长课堂分析已经稳定，除非经过长文本、校验器和导出安全验证。
- 不说声纹身份识别、底噪处理、自动说话人分段已经完成。
- 不说 vivo ASR/OCR provider 已接通，除非对应官方接口与真机验证完成。

## 3. 评委问到“为什么要端侧”

推荐回答：

端侧模型解决的是云端模型之外的可用性和隐私问题。课堂学习 App 会遇到弱网、断网、复习时的小问题解释，以及不希望上传的敏感材料。我们不会让端侧模型绕过证据链和校验器，它只作为 provider 链中的智能兜底，并且所有结果都会标注来源。

不该说：

- “端侧模型已经能完全替代云端。”
- “所有功能都可以离线跑大模型。”
- “端侧输出不需要校验。”

## 4. 评委问到“是不是套壳”

推荐回答：

ClassMate 不只是把模型接进聊天框。我们的核心是课堂资料进入 MaterialBundle 后，生成可验证知识点、证据链、微测、复习任务、需要多练和可导出报告。模型是增强层，不能替代证据定位、validator、LearningStore 和 Export 安全链。

## 5. 端侧适合展示的 proof

- Settings 端侧诊断：SDK、模型目录、init/generate 状态。
- provider path：Official BlueLM、OnDeviceBlueLM、LocalRule 的清晰标注。
- 弱网/离线下的兜底行为。
- StudyReport 不含密钥和完整模型交互。
- Ask This Lesson 缺证据时拒绝胡编。

## 6. 20 秒收尾话术

Stage 8A 的重点不是把每个按钮都变成模型调用，而是把 Official BlueLM、端侧 BlueLM 和 LocalRule 放进一个可解释、可验证、可兜底的学习系统。复赛版先保证云端主路径和学习闭环，决赛版再把端侧模型扩展到隐私模式和离线学习增强。
