# Ask This Lesson 风险说明

## 1. 模型可能胡编

风险：当学生提出超出课堂范围的问题时，模型可能给出流畅但没有课堂依据的答案。

缓解：

- 输出必须带 grounded / partial / not_found。
- 没有可定位证据时，不把答案标成 grounded。
- not_found 模板应明确“本节课资料中没有找到明确依据”。

答辩解释：

“我们不把通用知识回答直接当作课堂结论。ClassMate 更看重证据可追溯，而不是答案看起来有多完整。”

## 2. quote 可能被改写

风险：模型返回的 evidence quote 可能是课堂句子的改写，导致无法定位原文。

缓解：

- quote 定位失败时降级为 partial 或 not_found。
- 不伪造 segment，不把改写句当作原文证据。
- 可以显示“未找到可引用原文”的短提示。

答辩解释：

“如果模型改写了证据，我们宁愿降级，也不伪造原文。这一点和 ClassMate 的证据链原则一致。”

## 3. 证据检索可能漏召回

风险：课堂文本较长或同义表达较多时，系统可能没有召回最合适的证据片段。

缓解：

- grounded 答案优先使用可直接定位的短 quote。
- partial 答案允许说明“只找到部分相关依据”。
- 后续可接入文本向量或术语表增强召回，但复赛阶段不夸大。

答辩解释：

“漏召回比胡编更可控。我们会把它暴露成 partial，而不是隐藏成看似确定的答案。”

## 4. 问题超出课堂范围

风险：学生会问相关但课堂没讲的问题，比如从机器学习基本流程跳到 Transformer。

缓解：

- 保留 not_found 状态。
- 可给出“建议另开专题学习”的简短提示。
- 不输出大段课外解释，以免破坏课堂边界。

答辩解释：

“ClassMate 不是不懂这些知识，而是当前功能要回答‘这节课讲了什么’，不是替代所有课外搜索。”

## 5. local_only 能力有限

风险：local_only 只能基于已提取知识点和证据做简单回答，表达能力弱于云端模型。

缓解：

- local_only 必须明确标注 fallback。
- 回答可以短，但必须不越界。
- 不因能力有限而发起网络调用。

答辩解释：

“LocalFallback 是系统可用性和隐私边界的一层保护，不是为了替代官方模型主路径。”

## 6. 运行内容不能进日志

风险：调试时把完整模型输入、请求消息、供应商原始返回或内部思考文本写进日志，会造成隐私和合规问题。

缓解：

- 日志只允许短状态、provider、耗时、错误类型、证据数量等字段。
- UI debug 面板只展示脱敏摘要。
- 导出文件只保留学习内容和证据摘要。

答辩解释：

“我们把日志当成工程诊断，不当成内容存储。课堂原文和模型运行细节不会被完整写入日志。”

## 7. answer 是否可导出需脱敏

风险：Ask 答案如果进入导出报告，可能带出不应展示的运行内容。

缓解：

- 导出只包含 question、answer、groundedness、related knowledge points 和短证据摘要。
- 不导出内部运行上下文。
- 导出前可复用安全检查清单。

答辩解释：

“导出是给学生复习看的，不是给工程调试看的。因此只保留学习价值内容。”

## 8. 如何向评委解释边界

建议话术：

“Ask This Lesson 当前的核心价值不是开放域聊天，而是证据约束问答。我们故意把答案限制在本节课材料里：能回答就给证据，只能部分回答就标 partial，没有依据就 not_found。这样既能减少幻觉，也能和知识时间线、微测、复习任务连成学习闭环。”

## 9. 后续增强路线

- 用课程术语表提高证据检索召回。
- 用文本相似度做跨片段证据匹配。
- 接入多资料来源后，在证据里标注 transcript / slide OCR / blackboard OCR / manual note / imported text。
- 对导出问答记录增加安全检查。
