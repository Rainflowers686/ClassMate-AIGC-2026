# OCR / PPT / 板书资料导入复赛演示脚本

目标：5 分钟内展示 ClassMate 不只是粘贴课堂文本工具，而是已经为课件、板书、讲义等课堂资料接入做好产品壳和证据链准备。当前演示是“手动 OCR / 接口预留”，不声称真实 OCR 已完成。

## 0:00-0:40 为什么 PPT / 板书识别重要

讲解词：

“课堂里很多关键信息并不只出现在老师口头讲解里。公式、流程图、例题步骤、哈斯图、代码片段，常常写在 PPT 或板书上。如果只做录音总结，容易漏掉这些视觉材料。ClassMate 的目标是把课堂文本、课件 OCR、板书 OCR 和手动笔记融合成同一节课的资料包，再生成知识时间线、证据链、微测和复习任务。”

## 0:40-1:20 演示选择“课件图片 / PPT 截图”

操作：

1. 打开 Import Hub。
2. 找到“PPT / 课件图片”入口。
3. 指出当前状态是下一阶段 OCR。

讲解词：

“这里的入口不是伪装已经完成图片识别。当前版本可以记录资料来源，并允许用户粘贴识别后的文字；后续拿到官方 OCR 文档和接口后，就可以把图片识别结果接入同一条路径。”

## 1:20-1:55 粘贴 OCR 文字

操作：

1. 使用 `docs/testing/ocr_slide_blackboard_samples.md` 中的“高数课件 OCR：级数判别法”。
2. 将“可粘贴 OCR 文本”放入课堂资料输入区域。
3. 标记来源为“课件 OCR”或在演示中口头说明来源。

讲解词：

“这段文字像是从 PPT 截图中识别出来的，包含部分和、收敛必要条件、几何级数、p 级数等要点。它不是完整讲稿，但足够成为课堂证据来源。”

## 1:55-2:30 加入本节课资料

操作：

1. 点击加入本节课资料。
2. 如有资料列表，展示已加入资料条目。
3. 确认资料来源显示为课件 OCR / 板书 OCR / 导入文本之一。

讲解词：

“ClassMate 未来会把这些资料统一到 MaterialBundle。每一段资料都带来源标签，后面生成知识点和证据时，不只知道答案是什么，也知道证据来自课堂转写、课件 OCR 还是板书 OCR。”

## 2:30-3:10 与课堂文本一起生成 Timeline

操作：

1. 同时保留一段课堂文本或手动笔记。
2. 生成知识时间线。
3. 展示 Timeline 中出现来自 OCR 资料的知识点。

讲解词：

“这里不是单独总结一张图，而是把课件文字和课堂文本合并分析。最终结果仍进入原来的知识时间线和微测流程，不绕开 BlueLM、Compatible 或 LocalFallback 主体系。”

## 3:10-3:45 展示证据来源

操作：

1. 打开一个知识点的证据链。
2. 展示 evidence quote 和来源标签。

讲解词：

“评委可以看到，这个知识点不是凭空生成的。证据可以标注为课件 OCR 或板书 OCR。后续如果接入真实 OCR，也只是在资料输入层升级，证据链和校验逻辑仍然保留。”

## 3:45-4:20 Ask This Lesson 引用 OCR 来源

操作：

1. 进入 Ask This Lesson。
2. 提问：“通项趋于 0 能保证级数收敛吗？”
3. 展示回答引用 OCR 片段。

讲解词：

“Ask This Lesson 不只是聊天，它要从本节课资料里找依据。若 OCR 资料中包含‘不是充分条件’和‘调和级数发散’，系统就可以给出 grounded 回答；如果没有依据，就应该标注 partial 或 not_found。”

## 4:20-4:45 导出报告显示资料来源摘要

操作：

1. 导出学习报告。
2. 展示资料来源摘要：课堂文本、课件 OCR、板书 OCR。

讲解词：

“导出报告面向学生复习，不包含调试内容或敏感信息。它应该保留学习价值：课程标题、知识点、证据、题目、复习计划，以及资料来源摘要。”

## 4:45-5:00 诚实说明

讲解词：

“当前阶段是手动 OCR 和接口预留，不声称真实 OCR 已完成。我们优先把资料来源、证据链、问答引用、导出摘要这套学习结构打通。后续接入官方 OCR 后，替换的是识别入口，不需要推翻学习闭环。”
