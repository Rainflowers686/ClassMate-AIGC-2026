# Real Device Test Manual - 1.14.6 / 119

## A. 蓝心配置与诊断

1. 打开开发者设置，保存官方 BlueLM AppID/AppKey。
2. 点击连接测试。
3. 预期：状态只显示已配置、网络/鉴权/超时/返回空等分类，不显示 AppKey 或 Authorization 值。
4. 失败时：课程分析仍保留本地整理版，用户可重试或继续使用本地版。

## B. ASR 主路线

1. 进入资料导入页的课堂录音卡。
2. 预期：主按钮为“开始录音”或官方路线文案，不再承诺“开始录音并实时转写”。
3. 点击开始录音，停止并保存。
4. 若官方 ASR 已配置，点击“官方 ASR 转写录音”；若未配置，使用“粘贴转写文本”。
5. 系统实时转写只在可选按钮中出现，文案必须说明依赖机型和 ROM。

## C. 无系统语音识别服务

1. 在无系统 SpeechRecognizer 的设备上进入录音卡。
2. 预期：录音仍可保存；不会卡死在“没有语音识别服务”。
3. 预期：可以打开系统语音设置、仅录音、粘贴转写文本。

## D. 微测相关性

1. 导入混有“同学们注意、重点来了、大家记一下”的 OCR/文本材料。
2. 生成课程。
3. 预期：知识点和题目围绕学科概念；强调词不成为知识点或题干核心。
4. 做题后查看答案详解。
5. 预期：详解包含对应知识点、证据摘录、正确项理由和错误项辨析。

## E. 反馈即时优化

1. 对题目点击反馈“不准确”。
2. 预期：当前题被标记/移除，并出现基于同一知识点或证据的替换题。
3. 对知识点/证据反馈。
4. 预期：页面立即显示“需复核”或替代证据，不只显示 toast。
