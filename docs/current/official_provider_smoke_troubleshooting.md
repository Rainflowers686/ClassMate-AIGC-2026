# Official Provider Smoke Troubleshooting

## 2026-06-17 OCR Invalid URI

Observed old result:

- capability: `OCR`
- old status: `FAIL_NETWORK`
- actual root cause: invalid URI generated by smoke URL composition
- bad URI shape: host followed directly by `=classmate-smoke`
- request should be considered not sent

This was a smoke harness bug, not proof that the OCR official endpoint, AppID, or local AppKey failed. No endpoint value, key value, authorization value, cookie, or token is documented here.

## v5 Fix

The smoke harness now:

- joins `baseUrl` and `endpointPath` by trimming one trailing slash from the base and ensuring the path starts with `/`
- preserves endpoint paths that already include query strings
- appends smoke trace data as query parameters with `?` or `&`
- validates the final URL with `System.Uri.TryCreate`
- reports invalid composition as `FAIL_INVALID_URI`
- keeps `requestSent=False` until a request reaches `Invoke-WebRequest`

## Status Meanings

- `FAIL_INVALID_URI`: URL composition failed before a network request. Check `baseUrl` and `endpointPath` field names and shapes.
- `SKIPPED_ENDPOINT_MAPPING_MISSING`: no reliable endpoint mapping exists.
- `SKIPPED_CONFIG_MISSING`: endpoint may be mapped, but auth/config is missing.
- `SKIPPED_SEAM_ONLY`: code seam exists, but live provider request mapping is not confirmed.
- `FAIL_TIMEOUT`: request exceeded `-TimeoutSeconds`.
- `FAIL_NETWORK`: a request was attempted and failed outside the more specific categories.
