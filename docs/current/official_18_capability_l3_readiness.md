## 4. 横向收口要点
- **已用于学习闭环 / 真机已验证**：仅「大模型」「通用 OCR」（其余诚实标为 fallback / seam / config-required / experimental）。
- **自动本地兜底**：云端 + 端侧都失败且有可用输入时，自动进入「本地基础整理」（真实知识点/题目/证据），不是空安全占位。
- **实验性默认关闭**：图片生成、视频生成、同声传译。
- **仍需真机/云真机抽测**：大模型、OCR、长语音转写、端侧 3B、端侧文件（见上表「真机抽测=是」）。
- **端侧多模态**：保持「诊断 / 待验证 / 配置缺失」，不写成完整可用。

## 5. 真机抽测优先顺序（与主链一致）
1. 预置凭据/模型/权限（`scripts/qa/demo_device_provision.ps1`）→ 一次云端分析确认来源=云端蓝心。
2. 主链文本闭环：知识点 + 证据回溯 + Evidence-grounded Ask（无依据不编造）。
3. 练习闭环：微测 → 错题本 → 复习 → 掌握度 → 学习诊断。
4. OCR 真演示（唯一可台上证明的官方实时整合）。
5. 断网兜底：端侧 / 本地基础整理 / 安全占位 诚实分级。
6. Study Pack 导出（脱敏 + A4 + 8 段核对）。
7. 音频/方言（最后；只讲 artifact + 手动/导入转写，不讲自动听课超越听脑）。
