# Defense Narrative

## 一句话

ClassMate 不是普通聊天 App，而是一个证据绑定的学习闭环系统：资料 -> 知识点 -> 证据 -> 微测 -> 反馈 -> 复习 -> 精修导出。

## 为什么不是聊天壳

- 输入不是“问一句”，而是课堂资料。
- 输出不是一段回答，而是结构化学习资产。
- 每条结论尽量绑定证据。
- 答题反馈能进入错题、薄弱点和复习计划。
- 最终能导出成可打印学习包。

## 官方能力叙事

我们使用蓝心/官方能力作为增强主路径，但产品不依赖单点成功：

- 蓝心大模型优先。
- 端侧 3B optional fallback。
- 本地规则保证最低闭环。
- 官方 ASR/TTS 已代码接入，但真实网络仍需 AppKey 和真机验证。
- 系统 ASR/TTS 与手动输入保证课堂场景不中断。

## 竞品差异

- 不是只转写：转写后进入知识结构、微测、复习。
- 不是只刷题：题目来自自己的课堂资料。
- 不是只总结：证据回溯和反馈闭环是主线。
- 不是只导出：导出来自完整学习闭环。

## 风险回答

如果评委问“官方能力是否都已跑通”：

> 代码层已经接入关键官方能力，并为真机验证准备了 fallback 和 QA 脚本。我们不会把未配置或未联网验证的能力说成已真实跑通；比赛演示会使用核心学习闭环和已验证 fallback，官方网络能力按现场配置验证。
