# Learning Loop Architecture

ClassMate 的核心不是聊天，而是证据绑定学习闭环。

## 闭环阶段

1. **资料篮**：文本、图片、文档、录音、字幕进入同一资料入口。
2. **OCR/manual fallback**：图片成功项进入 OCR_IMAGE evidence；失败项可手动输入。
3. **知识结构**：生成课程主题、知识点、重点解释、易错点。
4. **Evidence**：每条知识点/题目尽量绑定来源；弱/无证据有三态。
5. **Quiz**：生成可答题微测；无模型题时有本地基础题。
6. **Feedback**：答题反馈进入错题、需复核、已复习和薄弱点。
7. **ReviewPlan**：结合错题、薄弱点、复习状态安排任务。
8. **Polished export**：普通导出始终可用；AI 精修导出主动触发。

## 证据类型

| 类型 | UI 展示 | 失败降级 |
| --- | --- | --- |
| TEXT | 原文片段 | 暂无可回溯证据 |
| OCR_IMAGE | 图片/缩略图 + OCR 文本 | 图片资产缺失时保留文本证据 |
| DOCUMENT | 文件名、页码/段落、片段 | 手动页文本 |
| AUDIO | 录音名、时间段、转写片段 | 播放定位待真机验证 |
| WEB | URL/摘要 | 浏览器 Intent，不抓取 API |

## 微测和反馈

- 答题前不显示答案。
- 提交后显示正确答案、解析、证据。
- 错题进入 WrongBook，关联 KnowledgePoint 和 ReviewPlan。
- 用户反馈不是只收集，能影响复习状态。
