同学们，今天我们讲 C++ 中的指针、引用、内存和函数参数传递。先从变量和地址说起。程序中的变量会存放在某一块内存中，每个内存位置都有地址。指针变量保存的是另一个对象的地址，因此指针的核心含义是“间接访问”。例如 int* p 表示 p 是指向 int 对象的指针，若 p 指向变量 x，那么 *p 就表示通过地址访问 x 的值。取地址运算符 & 用来获得变量地址，解引用运算符 * 用来访问指针指向的对象。

指针最容易出错的地方是空指针、野指针和悬空指针。空指针表示当前没有指向有效对象，现代 C++ 推荐使用 nullptr。野指针是未初始化的指针，它保存的地址不确定，解引用会导致未定义行为。悬空指针是曾经指向有效对象，但对象生命周期已经结束后仍然被使用的指针。写代码时要养成初始化指针、检查空指针、明确对象生命周期的习惯。

引用可以理解为对象的别名。声明 int& r=x 后，r 和 x 指向同一个对象，对 r 的修改就是对 x 的修改。引用必须在定义时初始化，初始化后不能再改绑到另一个对象。相比指针，引用语法更简洁，也更适合表达“函数需要修改实参”或“避免拷贝大对象”的意图。const 引用尤其常见，例如 void print(const string& s)，它避免拷贝，同时保证函数内部不能修改传入对象。

接着看函数参数传递。值传递会把实参复制一份给形参，函数内部修改形参不会影响外部实参。指针传递把地址传给函数，函数可以通过解引用修改外部对象，但调用处需要传地址，且要处理空指针风险。引用传递让形参成为实参别名，函数内部可以直接修改外部对象。const 引用传递适合只读访问大对象，既避免拷贝，又保持接口安全。

内存方面，C++ 对象可能位于栈、自由存储区或静态存储区。局部变量通常在栈上，离开作用域自动销毁。使用 new 创建的对象在自由存储区，需要 delete 释放；但现代 C++ 更推荐使用智能指针和 RAII，让资源生命周期由对象自动管理。unique_ptr 表示独占所有权，shared_ptr 表示共享所有权。理解指针和引用，不只是记语法，更重要的是理解对象生命周期、所有权和函数接口意图。做题或写程序时，先判断是否需要修改实参，再选择值传递、指针传递、引用传递或 const 引用传递。
