# Sample Lessons README

这些文件是 ClassMate 真机测试和 BlueLM 长文本验证用的标准课堂文本。每个 `.txt` 文件只包含可直接复制进 App 的课堂讲稿，不包含说明、表格或测试备注。

## 推荐测试顺序

1. Import Hub：复制一个 `.txt` 文件全文进课堂文本输入框。
2. Analyze：使用当前 provider profile 生成结果。
3. Timeline：检查是否有 6-8 个知识点，标题是否是概念而不是课堂套话。
4. Evidence：点开 2-3 个知识点，检查证据是否能回到原文句子。
5. Quiz：答 2-3 道题，检查解释是否引用课程概念。
6. Review：检查是否生成 ReviewTask。
7. History / Course Library：确认课程记录可回看。
8. Export：导出报告并检查不含敏感字段。

## 预期知识点方向

每个样例都应能抽取 6-8 个知识点。合理输出应该覆盖：

- 定义型知识点。
- 判别条件或公式。
- 易错点。
- 方法步骤。
- 典型例子或应用场景。
- 概念之间的区别。

## 预期微测方向

每个样例适合生成 5-8 道微测，题型建议覆盖：

- 概念理解。
- 判断题。
- 应用题。
- 错因分析。
- 迁移理解。

## 如果模型生成结果偏少

优先检查：

1. 是否复制了完整文本，而不是只复制一小段。
2. 当前 model 的 `maxTokens` 是否过低。
3. 当前 provider 是否 fallback 到本地规则。
4. 课堂文本是否有足够分段和明确概念。

建议调整：

- 对真实 BlueLM 测试，保持课堂文本在 1200-1800 中文字左右。
- 若知识点少于 5 个，可尝试提高 `maxTokens` 到 1800-2200。
- 若输出过长或超时，可控制在 1200-1500 中文字，并保持 temperature 较低。
- 不要用极短文本判断长课堂能力。

## 安全要求

- 样例不包含个人隐私。
- 样例不包含真实密钥。
- 不要把本地配置内容复制到课堂文本框。
- 导出后仍需搜索敏感字段，确保报告只包含学习业务数据。
