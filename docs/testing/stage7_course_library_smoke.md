# Stage 7D Course Library Smoke

目标：验证课程库、首页引导和 Settings 能力路线图，不测试 Review/Practice 的内部规则。

## 1. 首页首次引导

- 操作：清空或使用没有历史课程的测试环境，启动 App 到首页。
- 预期：显示“三步开始”卡片，包含“导入课堂资料 / 生成知识时间线 / 开始复习与练习”。
- 截图位置：首页首屏。
- 失败排查：检查 HomeScreen 是否仍按 `ui.history.isEmpty()` 显示引导。

## 2. 首页引导弱化

- 操作：导入任一示例课堂并生成一次时间线，返回首页。
- 预期：“三步开始”不再长期占据首页，首页仍保留今日状态、导入、Live、最近课程和复习摘要。
- 截图位置：首页首屏。
- 失败排查：检查历史记录是否成功写入。

## 3. 课程库搜索

- 操作：进入“课程库”，在搜索框输入课程标题、知识点标题、科目或来源标签，例如“级数”“BlueLM”“本地兜底”。
- 预期：课程列表按关键词收窄；找不到时显示“没有找到课程 / 清除筛选”。
- 截图位置：课程库搜索框和结果列表。
- 失败排查：确认该课程已经生成 History 记录，且知识点标题存在于分析结果。

## 4. 课程库筛选

- 操作：依次点击“全部 / 有待复习 / 有薄弱点 / 最近学习 / 官方 BlueLM / 本地兜底”。
- 预期：列表按状态变化；没有命中时出现清除筛选空态。
- 截图位置：筛选 chip 和列表。
- 失败排查：有薄弱点需要先在 Quiz/Review 中产生答错、太难、需要多练或证据复核信号。

## 5. 课程库排序

- 操作：切换“最近学习 / 知识点数量 / 待复习数量 / 课程标题”。
- 预期：课程卡片顺序变化，卡片统计不丢失。
- 截图位置：排序 chip 与前三张课程卡。
- 失败排查：确认测试数据至少有两门课程，否则排序效果不明显。

## 6. 课程详情

- 操作：点击任一课程卡。
- 预期：进入课程详情，可继续打开时间线、Ask、Quiz、复习计划、资料来源和导出中心。
- 截图位置：课程详情顶部。
- 失败排查：确认 HistoryRecord 中保留了 session/result。

## 7. Settings 能力路线图

- 操作：进入设置，查看“能力路线图”。
- 预期：分组为“已支持 / 实验模式 / 待接入 / 暂缓”；ASR/OCR/声纹等未完成能力不被夸大。
- 截图位置：Settings 能力路线图卡片。
- 失败排查：检查 SettingsScreen 文案。

## 8. 复赛演示话术

课程库不是单纯历史列表，而是按课程聚合学习资产：每门课能看到最近学习时间、课堂次数、知识点、微测和待复习任务。搜索和筛选帮助评委快速看到 BlueLM 正常生成的课程、LocalFallback 兜底课程，以及已经产生薄弱点的课程。
