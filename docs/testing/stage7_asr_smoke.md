# Stage 7A · 实时转写实验（系统 ASR）真机冒烟

实验模式：使用系统语音识别（Android SpeechRecognizer）边说边转。**不保存原始音频、不上传音频、不后台录音、不做声纹识别。** 效果依设备而定，部分云真机/无 Google 服务设备可能不支持。

## 最短冒烟路线

1. 打开 App → 课堂伴学 Live。
2. 找到“实时转写实验”卡片，确认免责文案（不保存原始音频 / 不后台录音）。
3. （可选）选择说话人：教师 / 学生 / 未知。
4. 点“开始实时转写”。
   - 首次会弹出麦克风权限请求 —— 这是用户主动点击后才请求。
   - 授权后状态应变为“识别中”。
5. 对着麦克风说一小段话（一两句），停顿一下让系统出 final 结果。
6. 观察：
   - “识别中：…” 显示临时(partial)文本；
   - 出 final 后，“已确认 N”增加，下方“已确认片段”出现该句（带时间与说话人标签）。
7. 点“停止”。
8. 点“转为手动编辑”进入转写编辑器，可改文本 / 说话人 / 时间，保存进资料篮；
   或直接点“生成知识时间线”。
9. 生成后进入课程详情 → 知识时间线 / 微测 / 复习 / 导出，确认实时转写内容已进入分析。
10. 导出报告，确认资料来源含“实时转写”，正文 marker 形如 `[实时转写 00:01:05-00:01:12 · 教师]`。

## 兜底验证

- **设备不支持**：点“开始实时转写”后状态为“不支持”，提示改用手动转写或导入字幕；手动片段功能仍可用。
- **拒绝权限**：拒绝麦克风后状态为“待授权麦克风”，提示仍可手动记录或导入转写稿。
- **识别出错 / 无网络**：状态变“出错”并给出提示，已确认片段不丢失，可继续手动或停止。
- **partial 不稳定**：只有 final 结果进入分析，partial 仅显示。

## 注意

- 系统 ASR 是否走云端由 OEM 识别器决定，ClassMate 不上传音频、不接 vivo ASR endpoint。
- 自动说话人分段 / 声纹身份识别 / 底噪处理仍为占位，未实现。
- 不保证所有云真机可用；不可用时以手动转写 + 字幕导入为主路径。
