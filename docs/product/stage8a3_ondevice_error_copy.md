# Stage 8A-3 端侧 SDK 错误文案库

> 全部中文，适合 Settings UI / 诊断页面 / Toast / Dialog 使用。
> 每个错误包含：用户文案、调试标签、建议操作、是否阻塞复赛演示。

---

## SDK 检测错误

### 1. SDK 未发现

- **用户文案**：未检测到端侧智能 SDK。端侧模型不可用，已自动使用云端模型。
- **调试标签**：`SDK_MISSING`
- **建议操作**：检查 `app/libs/llm-sdk-release.aar` 是否存在；检查 AAR 是否包含 `classes.jar`。
- **阻塞复赛**：否（可回退云端）

### 2. SDK 版本不匹配

- **用户文案**：端侧 SDK 版本与应用不兼容。请更新应用或 SDK 后重试。
- **调试标签**：`SDK_VERSION_MISMATCH`
- **建议操作**：检查 AAR 构建版本与对接代码期望的 SDK 版本；用 javap 确认类和方法签名一致。
- **阻塞复赛**：**是**（端侧完全不可用，需要重新构建 AAR）

## LlmConfig 错误

### 3. LlmConfig 字段缺失

- **用户文案**：端侧模型配置异常，缺少必要参数。请联系技术支持。
- **调试标签**：`LLMCONFIG_FIELD_MISSING`
- **建议操作**：检查 `LlmConfig` 构造代码中是否遗漏必填字段（modelPath、nPredict 等）；用 javap 确认 AAR 中的字段名。
- **阻塞复赛**：**是**（init 直接失败）

### 4. 模型路径不存在

- **用户文案**：端侧模型文件未找到。请下载模型文件后重试。
- **调试标签**：`MODEL_PATH_NOT_FOUND`
- **建议操作**：检查 `LlmConfig.modelPath` 指向的目录是否存在；确认模型文件已下载到指定位置。
- **阻塞复赛**：**是**（init 直接失败）

### 5. 模型路径不可读

- **用户文案**：端侧模型文件无法读取。请检查存储权限或重新下载模型。
- **调试标签**：`MODEL_PATH_UNREADABLE`
- **建议操作**：检查应用是否有存储读取权限；检查模型文件权限；检查 SELinux 上下文（Android 11+）。
- **阻塞复赛**：**是**（init 直接失败）

## LlmManager 错误

### 6. LlmManager 方法缺失

- **用户文案**：端侧 SDK 接口异常。请确认 SDK 版本是否正确。
- **调试标签**：`LLMMANAGER_METHOD_MISSING`
- **建议操作**：用 javap 检查 `LlmManager` 类的方法列表；确认 `callVit(byte[], int, int)` 和 `generate(String, TokenCallback)` 是否存在。
- **阻塞复赛**：**是**（reflect bridge 无法工作）

### 7. TokenCallback 签名不匹配

- **用户文案**：端侧 SDK 回调接口异常。请确认 SDK 版本。
- **调试标签**：`TOKENCALLBACK_SIGNATURE_MISMATCH`
- **建议操作**：确认 `onComplete()` 无参数（不是 `onComplete(LlmStats)`）；确认 `onError(int, String)` 签名正确。
- **阻塞复赛**：**是**（回调无法正常接收）

## 初始化错误

### 8. 初始化失败

- **用户文案**：端侧模型启动失败。已切换至云端模型，您可以继续使用。
- **调试标签**：`INIT_FAILED`
- **建议操作**：检查 init 返回的具体错误码；检查 native library 加载状态；检查设备内存是否充足。
- **阻塞复赛**：否（可回退云端，但影响端侧 proof）

### 9. Native Library 加载失败

- **用户文案**：端侧模型依赖库加载失败。当前设备可能不完全支持端侧推理。
- **调试标签**：`NATIVE_LIBRARY_LOAD_FAILED`
- **建议操作**：检查 AAR 中是否包含对应 ABI 的 .so 文件；检查设备 SoC 架构（arm64-v8a/armeabi-v7a）。
- **阻塞复赛**：**是**（端侧完全不可用）

### 10. NPU/APU 不可用

- **用户文案**：设备 AI 加速芯片不可用。端侧推理将以较低性能运行或不可用。
- **调试标签**：`NPU_APU_UNAVAILABLE`
- **建议操作**：检查设备是否支持 NPU/APU；检查 `LlmConfig.npuPower` 配置是否合理；尝试 CPU fallback。
- **阻塞复赛**：否（可能影响推理速度但不一定阻塞）

## 推理错误

### 11. Generate 超时

- **用户文案**：端侧模型响应超时。已切换至云端模型，您可以继续使用。
- **调试标签**：`GENERATE_TIMEOUT`
- **建议操作**：延长超时阈值或检查 prompt 长度是否合理；检查设备是否处于高温 throttling 状态。
- **阻塞复赛**：否（可回退云端）

### 12. onError 错误码 — 通用

- **用户文案**：端侧模型处理出错（错误码：{code}）。已切换至云端模型。
- **调试标签**：`ONERROR_CODE_{code}`
- **建议操作**：根据 error code 查询 SDK 文档；常见错误码：1=参数错误、2=模型未加载、3=推理失败、4=内存不足。
- **阻塞复赛**：取决于错误码。1/2/3 为**是**；4 为否（可重试）

### 13. onToken 无输出

- **用户文案**：端侧模型未返回结果。请尝试缩短问题后重试。
- **调试标签**：`ONTOKEN_NO_OUTPUT`
- **建议操作**：确认 `onComplete()` 是否被触发；检查 prompt 是否为空或过长；检查是否有 interrupt 被意外调用。
- **阻塞复赛**：否（需排查，但不放弃）

### 14. onComplete 未触发

- **用户文案**：端侧模型响应未正常结束。请重试或切换至云端模型。
- **调试标签**：`ONCOMPLETE_NOT_TRIGGERED`
- **建议操作**：检查 generate 线程是否被阻塞；检查 TokenCallback 引用是否被 GC 回收；增加超时机制。
- **阻塞复赛**：否（可回退云端）

## 多模态错误

### 15. callVit 不存在

- **用户文案**：端侧多模态功能不可用。当前 SDK 版本可能不支持视觉编码。
- **调试标签**：`CALLVIT_NOT_FOUND`
- **建议操作**：用 javap 确认 `LlmManager` 中是否存在 `callVit` 方法；确认 AAR 是否包含多模态模块。
- **阻塞复赛**：否（多模态为实验性功能，纯文本优先）

### 16. callVit 返回非 0

- **用户文案**：图片处理失败。请尝试其他图片或使用云端模型处理。
- **调试标签**：`CALLVIT_FAILED_CODE_{code}`
- **建议操作**：检查输入 bitmap 尺寸；确认 RGB byte 数组长度 = width × height × 3；检查设备 NPU 状态。
- **阻塞复赛**：否（多模态为实验性功能）

### 17. 多模态 Prompt 缺少 Image Token

- **用户文案**：端侧多模态请求格式异常。请稍后重试。
- **调试标签**：`MULTIMODAL_PROMPT_NO_IMAGE_TOKEN`
- **建议操作**：确认多模态 prompt 中包含 image token 占位符；检查 prompt 模板是否正确。
- **阻塞复赛**：否（多模态为实验性功能）

## 生命周期错误

### 18. Release 后再次调用失败

- **用户文案**：端侧模型会话已结束。请重新开始对话。
- **调试标签**：`RELEASED_INSTANCE_CALLED`
- **建议操作**：确认 release() 调用时机；release 后需要重新 init 才能使用；建议使用状态机管理生命周期。
- **阻塞复赛**：否（正确使用即可避免）

---

## 权限错误

### 19. 权限不足

- **用户文案**：缺少必要权限，端侧模型无法运行。请在系统设置中授予存储权限。
- **调试标签**：`PERMISSION_DENIED`
- **建议操作**：引导用户前往系统设置开启存储权限；确认 AndroidManifest 中权限声明正确。
- **阻塞复赛**：**是**（无存储权限时模型文件无法读取）

---

## Fallback 状态

### 20. 本地兜底已启用

- **用户文案**：当前使用离线规则响应。连接网络或启用端侧模型以获得更好的体验。
- **调试标签**：`LOCALRULE_FALLBACK_ACTIVE`
- **建议操作**：这是正常状态（兜底），不是错误。检查端侧和云端 Provider 为何不可用。
- **阻塞复赛**：否（这是正常 fallback 行为）
