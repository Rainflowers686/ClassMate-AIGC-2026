# Stage 8A-7 Settings Diagnostic Copy Review

> Settings 端侧模型诊断页面文案审查表。
> 面向复赛演示和用户可理解性。所有推荐文案为中文。

---

## 审查总原则

1. **诚实**：不夸大端侧能力，不隐瞒限制。
2. **清晰**：不堆技术术语（除必须外），普通大学生能看懂。
3. **安全**：暗示"不上传"、"本地处理"、"不保存原图"等隐私保证文字必须有对应工程实现，**不虚假承诺**。
4. **可截图**：文案应在截图 proof 中清晰可见。

---

## 逐条审查

### 1. SDK 已发现

| 项目 | 内容 |
|------|------|
| **推荐文案** | "BlueLM 端侧 SDK 已就绪" |
| **禁止文案** | "SDK 加载成功"（可能混淆：SDK 存在 ≠ native 加载成功） |
| **原因** | "已就绪"仅表示 AAR 可被 reflection 加载，不保证 init 成功 |
| **适合截图吗** | ✅ 是 — `SETTINGS_ONDEVICE_SECTION` proof |

### 2. SDK 未发现

| 项目 | 内容 |
|------|------|
| **推荐文案** | "未检测到端侧 SDK，端侧模型不可用" |
| **禁止文案** | "SDK 缺失"（太技术），"端侧功能已禁用"（暗示用户可启用） |
| **原因** | 明确告知不可用，不暗示用户可以自行解决 |
| **适合截图吗** | ✅ 是 — `CLOUD_DEVICE_FAILURE` 类 proof |

### 3. 模型路径

| 项目 | 内容 |
|------|------|
| **推荐文案** | "模型路径：/sdcard/1225/"（如实显示配置值） |
| **禁止文案** | "模型路径有效"/"模型路径无效"（太武断，可能误判） |
| **原因** | 只显示路径，不自行判断有效性 |
| **适合截图吗** | ✅ 是 — `SETTINGS_MODEL_PATH` proof |

### 4. 初始化成功

| 项目 | 内容 |
|------|------|
| **推荐文案** | "初始化成功 — BlueLM 3B 已就绪" |
| **禁止文案** | "模型加载完成，可以使用"（可能暗示模型已加载到内存；init 成功 ≠ 模型完全加载） |
| **原因** | "初始化成功"是 SDK 回调的标准表述，简洁准确 |
| **适合截图吗** | ✅ 是 — `SETTINGS_TEXT_INIT_SUCCESS` proof |

### 5. 初始化失败

| 项目 | 内容 |
|------|------|
| **推荐文案** | "初始化失败（错误码: {code}），请检查模型路径" |
| **禁止文案** | "初始化失败，请重试"（重试通常无效，给错误码更有用） |
| **原因** | 附错误码方便调试，不暗示"重试即解决" |
| **适合截图吗** | ✅ 是 — bug triage 用 |

### 6. 生成成功

| 项目 | 内容 |
|------|------|
| **推荐文案** | 直接显示 AI 输出文本，状态栏显示"已完成" |
| **禁止文案** | "AI 生成成功"（冗余），"答案由 BlueLM 3B 提供"（可能不准确） |
| **原因** | 输出文本即是最好的成功证明 |
| **适合截图吗** | ✅ 是 — `SETTINGS_TEXT_GENERATE_SUCCESS` proof |

### 7. 生成失败

| 项目 | 内容 |
|------|------|
| **推荐文案** | "生成中断（错误码: {code}），请查看日志" |
| **禁止文案** | "生成失败，请稍后重试"（模糊），"AI 无法回答"（责任归因错误） |
| **原因** | 附错误码，指向日志而非"稍后重试" |
| **适合截图吗** | ✅ 是 — bug triage 用 |

### 8. 多模态支持

| 项目 | 内容 |
|------|------|
| **推荐文案** | "多模态编码（实验功能）" |
| **禁止文案** | "支持图片理解"、"拍照问答"、"图片转文字"（夸张） |
| **原因** | 当前仅 VIT 编码 diagnostic，未接入学习主链，不是"图片理解"功能 |
| **适合截图吗** | ✅ 是 — `SETTINGS_MULTIMODAL` proof |

### 9. VIT 编码成功

| 项目 | 内容 |
|------|------|
| **推荐文案** | "图像编码成功（返回码: 0），可继续文本生成" |
| **禁止文案** | "图片识别完成"、"视觉特征提取成功"（太产品化） |
| **原因** | 说清楚是编码成功且返回码 0，功能边界明确 |
| **适合截图吗** | ✅ 是 — `SETTINGS_MULTIMODAL_CALLVIT_SUCCESS` proof |

### 10. VIT 编码失败

| 项目 | 内容 |
|------|------|
| **推荐文案** | "图像编码失败（返回码: {code}），请勿继续生成" |
| **禁止文案** | "图片格式不支持"（未确认原因时不要乱归因） |
| **原因** | 附返回码，明确"不要继续 generate" |
| **适合截图吗** | ✅ 是 — bug triage 用 |

### 11. 本地规则兜底

| 项目 | 内容 |
|------|------|
| **推荐文案** | "当前使用本地规则兜底 — AI 模型未连接" |
| **禁止文案** | "离线模式"（混淆：LocalRule 不是离线版 AI） |
| **原因** | 明确区分 LocalRule（规则）和 AI（模型），避免用户以为 AI 仍在工作 |
| **适合截图吗** | ✅ 是 — `FALLBACK_LOCAL_RULE` proof |

### 12. 断网仍可尝试端侧模型

| 项目 | 内容 |
|------|------|
| **推荐文案** | "网络不可用 — 仍可尝试端侧模型（BlueLM 3B 本地推理）" |
| **禁止文案** | "断网也能用 AI"（把端侧模型说成通用 AI） |
| **原因** | 端侧模型能力有限（3B），不是"完整的 AI" |
| **适合截图吗** | ✅ 是 — 断网 proof 用 |

### 13. 不上传图片

| 项目 | 内容 |
|------|------|
| **推荐文案** | "图片在本地编码，不会上传云端" |
| **禁止文案** | "绝对不上传你的任何数据"（太绝对，且 range 超出图片范围） |
| **原因** | 明确范围：图片本地编码；不扩展到所有数据 |
| **适合截图吗** | ✅ 是 — 隐私 proof 用 |

### 14. 不保存原图

| 项目 | 内容 |
|------|------|
| **推荐文案** | "编码完成后自动丢弃原图" |
| **禁止文案** | "你的图片不会被存储"（「不保存」≠「不存储」——编码过程中有内存占用） |
| **原因** | 准确表述：编码后丢弃，不持久化 |
| **适合截图吗** | ⚠️ 文字说明，截图不易证明（需代码审查配合） |

### 15. 实验功能说明

| 项目 | 内容 |
|------|------|
| **推荐文案** | "端侧模型为实验功能，可能在部分设备上不可用。云端 BlueLM 是学习助手的默认模型。" |
| **禁止文案** | "端侧 AI 正在测试中，很快会上线"（给虚假时间线） |
| **原因** | 诚实标注实验性质和设备兼容性差异；引导用户使用云端主路径 |
| **适合截图吗** | ✅ 是 — Settings 诊断页整体截图 |

## 通用禁止事项

| 禁止文案模式 | 原因 |
|-------------|------|
| "智能"、"AI 大脑" | 夸大 3B 模型能力 |
| "保护你的隐私" | 过于宽泛的隐私承诺，需具体说明 |
| "完全离线 AI" | "完全"一词风险大——model init 可能仍需要一次网络校验 |
| "实时" | 端侧推理不是实时 |
| "准确率 90%" 等数字 | 无 benchmark 数据支撑 |
| "即将支持" | 不给时间线承诺 |
| "业界领先" | 无对比依据，复赛评委可能追问 |

## 截图建议汇总

| 截图 proof 类型 | 涉及文案 | 推荐截图位置 |
|----------------|---------|-------------|
| `SETTINGS_ONDEVICE_SECTION` | SDK 已发现、模型路径、实验功能说明 | Settings 端侧区域全景 |
| `SETTINGS_TEXT_INIT_SUCCESS` | 初始化成功 | init 按钮区域 + 状态 |
| `SETTINGS_TEXT_GENERATE_SUCCESS` | AI 输出文本 | input + output 框 |
| `SETTINGS_MULTIMODAL_CALLVIT_SUCCESS` | VIT 编码成功 | callVit 按钮 + 返回码 |
| `SETTINGS_MULTIMODAL_GENERATE_SUCCESS` | 多模态生成输出 | input + output（含图像提示） |
| `FALLBACK_LOCAL_RULE` | 本地规则兜底 | LocalRule 回复界面 |
| 断网 proof | 断网仍可尝试端侧模型 | 飞行模式 + 端侧模型可用提示 |
