# Stage 8C 用户价值地图

> 五类用户场景的端到端价值分析：大学生课堂、考研/考公备考、在职学习/培训、教师备课/复盘、弱网/隐私/本地学习场景。
> 每类用户覆盖：痛点、输入资料、ClassMate 处理链路、输出结果、端侧能力价值、复习闭环价值。

---

## 一、大学生（在校课堂学习）

### 痛点
- 课堂笔记零散，课后回顾不知道重点在哪
- 课件 PPT 和板书内容难以整理成结构化笔记
- 不知道是否真正理解了知识点（缺少自测）
- 期末复习时面对大量资料无从下手

### 输入资料
- 课堂文本笔记（粘贴/导入）
- 板书拍照
- 课件 PPT 截图
- 课堂录音转写稿（Live ASR 实验模式）
- 教材 Markdown 笔记

### ClassMate 处理链路
```
文本/图片/转写稿 → MaterialBundle(统一来源标记)
  → 云端 BlueLM CourseAnalysis → Knowledge Timeline(知识点+证据)
  → Quiz(微测) → Review(自适应复习) → Practice(搜索建议)
  → Report(可打印学习报告)
```

### 输出结果
- 结构化知识点时间线（重要性/难度分级）
- 每个知识点的原文证据引用
- 自动生成微测题目（选择/判断）
- 自适应复习任务列表（薄弱点优先）
- 可打印 PDF 学习报告

### 端侧能力价值
- **图书馆/自习室**：校园 Wi-Fi 不稳定时，端侧仍可做课程分析和问答
- **课间快速提问**：低延迟本地推理，不必等网络
- **图片理解**：拍板书/课件 → 端侧多模态草稿 → 确认后进分析

### 复习闭环价值
- 不是听完课就结束，课后可以自测、复习、练习
- Evidence 可追溯：知道知识点具体来自课堂哪段内容
- Review 追踪薄弱点：针对性复习而非从头翻笔记
- Report 可打印：线下复习有完整资料

## 二、考研 / 考公 / 证书备考者

### 痛点
- 学习资料量大且分散（教材、网课笔记、真题、错题本）
- 缺少系统化的自测和错题追踪机制
- 碎片时间（通勤、排队）想复习但信号差
- 不知道薄弱环节在哪，花了大量时间在已掌握的内容上

### 输入资料
- 网课笔记（Markdown/文本）
- 教材重点章节文本
- 培训资料截图
- 错题拍照
- 手动整理的考点列表

### ClassMate 处理链路
```
多来源资料 → MaterialBundle
  → CourseAnalysis → 考点级知识点 Timeline
  → Quiz(高频自测) → Review(错题追踪+需要多练)
  → Practice(搜索词 → 外部题库练习)
  → Report(阶段性复习档案)
```

### 输出结果
- 按考点组织的知识点（非简单摘要）
- 微测题目自动生成（选择题/判断题）
- "需要多练"考点列表（答错追踪）
- 搜索词卡片引导至外部练习资源
- 阶段性备考报告（可打印）

### 端侧能力价值
- **通勤碎片时间**：地铁/公交上信号差，端侧仍可问答和查看复习建议
- **错题隐私**：个人薄弱考点不过网络
- **图片错题**：拍纸质错题 → 端侧理解 → 录入复习

### 复习闭环价值
- 自测 → 发现薄弱 → 针对性练习 → 再次自测的循环
- Practice 搜索词引导找到合适的外部练习资源（不爬取平台内容）
- Report 可做阶段性备考档案：记录已掌握和待加强的考点

## 三、在职学习者（业余充电 / 企业培训）

### 痛点
- 学习时间碎片化，无法长时间专注
- 公司培训材料（PPT、讲义）缺乏结构化整理
- 培训后缺少自测手段，不知道学到了什么
- 学习内容可能涉及商业敏感信息，不想上传云端

### 输入资料
- 公司培训 PPT 文本/截图
- 行业报告/白皮书摘要
- 在线课程笔记
- 会议/讲座手动记录
- 职业资格证书备考资料

### ClassMate 处理链路
```
敏感培训资料 → 可选端侧优先分析(不上传)
  → CourseAnalysis(云端或端侧) → Timeline
  → Ask(基于资料的问答) → Quiz → Review → Practice → Report
```

### 输出结果
- 培训内容结构化整理（知识点+证据）
- 自测题目用于培训效果验证
- 复习提醒和薄弱点追踪
- 可打印培训总结报告

### 端侧能力价值
- **商业敏感内容**：端侧分析数据不离开设备，适合企业内部培训资料
- **晚间学习**：网络不限速时走云端，网络差时走端侧
- **个人学习隐私**：职业发展方向相关的学习内容不经过网络

### 复习闭环价值
- 培训不只是听一次，课后可以反复自测和复习
- 多个培训课程可在 Course Library 中统一管理
- Report 可作为培训成果证明

## 四、教师 / 培训讲师（备课与复盘）

### 痛点
- 备课资料整理耗时，需要从多来源（教材、论文、PPT）提炼知识点
- 难以快速评估学生对知识点的掌握情况
- 课后需要为学生提供结构化的复习材料
- 缺乏方便的工具生成课堂知识点提纲

### 输入资料
- 教材章节文本
- 参考文献摘要
- 往届课件/PPT 文本
- 课堂录音转写（用于复盘改进）
- 学生常见问题记录

### ClassMate 处理链路
```
备课资料 → MaterialBundle → CourseAnalysis
  → 知识点 Timeline(可编辑和调整)
  → Quiz 题目(可筛选和修改作为随堂测验)
  → Report(可打印分发给学生)
```

### 输出结果
- 课堂知识点提纲（含重要性和难度标注）
- 微测题目池（供随堂测验或作业使用）
- 可打印的学习资料（分发给学生）
- 课堂复盘分析（基于转写稿）

### 端侧能力价值
- **备课隐私**：未发布的课程资料在本地分析，不上传
- **教室场景**：课间可快速分析上一节课的录音转写

### 复习闭环价值
- 快速将教学内容转化为学生可用的复习资料
- 微测题目可直接用于课堂测验或作业
- Report 可作为学生课后必读材料

## 五、弱网 / 隐私 / 本地学习场景

### 痛点
- 地铁、飞机、偏远地区没有稳定网络
- 学习内容涉及个人隐私（如医疗、法律学习资料）
- 不想学习数据被上传到任何云服务
- 网络限速或按流量计费环境下无法使用云端 AI

### 输入资料
- 离线文本笔记
- 本地图片/拍照
- 离线 TXT/Markdown 文件
- 手动整理的资料

### ClassMate 处理链路
```
离线资料 → 端侧 BlueLM 3B CourseAnalysis(全程本地)
  → Timeline(来源: 端侧蓝心)
  → Ask(端侧本地问答) → Quiz/Review(端侧建议) → 本地 Report
```

### 输出结果
- 本地生成的课程分析（质量受 3B 模型限制，但可离线使用）
- 端侧问答结果（标注端侧来源）
- 本地复习建议
- 本地生成的报告（不上传）

### 端侧能力价值（此场景核心）
- **完全离线可用**：飞行模式下完成课程分析和问答
- **数据不离开设备**：所有推理在本地，适合隐私敏感场景
- **零网络成本**：不消耗移动流量
- **诚实边界**：端侧结果标注端侧蓝心，用户知道质量差异；双模型不可用时标注安全占位

### 复习闭环价值
- 即使完全离线，仍可走通"输入→分析→问答→复习"的本地闭环
- 网络恢复后，可以选择将课程同步到云端（如果用户愿意）
- 安全占位机制保证离线失败时不会生成假结果

## 跨场景端侧能力价值总结

| 场景 | 端侧主价值 | 质量预期 | 是否上传 |
|------|-----------|---------|---------|
| 大学生课堂 | 图书馆/自习弱网可用 | 中（简单课程好，复杂课程降级） | 可选走云端 |
| 考研备考 | 碎片时间离线问答 | 中（概念题好，论述题降级） | 可选走云端 |
| 在职学习 | 商业敏感内容本地分析 | 中（依赖课程复杂度） | 建议端侧 |
| 教师备课 | 未发布内容本地处理 | 中（可人工校对） | 可选走云端 |
| 弱网/隐私 | 完全离线闭环 | 中（3B 模型限制，诚实标注） | 不上传 |
