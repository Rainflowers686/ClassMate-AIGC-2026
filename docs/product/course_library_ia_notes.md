# Course Library IA Notes

## 目标

课程库承担“学习资产入口”的角色，不再只是最近历史记录列表。它帮助用户按课程回看知识时间线、证据链、微测、复习计划、导出报告和资料来源。

## 信息结构

- 首页：只显示今日概览、导入资料、Live Companion、最近一节课和复习摘要。
- 课程库：显示课程卡片、搜索、筛选、排序。
- 课程详情：承载时间线、Ask This Lesson、Quiz、复习计划、资料来源和导出中心入口。
- Review：继续负责今日任务、薄弱点和反馈，不把课程库当成错题本。

## 搜索范围

课程库搜索覆盖：

- 课程标题
- 知识点标题
- 科目
- provider 标签，例如 BlueLM official、Compatible demo、Local fallback

搜索不调用模型，不读取密钥，不扫描日志，也不读取本地配置文件内容。

## 筛选范围

- 全部：显示所有课程。
- 有待复习：来自课程聚合中的待复习任务计数。
- 有薄弱点：由已有 learning snapshot 推导，包含答错、太难、需要多练或证据复核信号。
- 最近学习：优先显示最近一周或最近窗口内课程。
- 官方 BlueLM：最近 provider 为 BlueLM official 的课程。
- 本地兜底：fallback 或 Local fallback 课程。

## 排序

- 最近学习：默认排序。
- 知识点数量：适合展示课程内容丰富度。
- 待复习数量：适合展示学习闭环。
- 课程标题：适合快速定位。

## 首页首次引导

无课程时显示“三步开始”：

1. 导入课堂资料。
2. 生成知识时间线。
3. 开始复习与练习。

已有课程后引导自动弱化，避免首页变成长说明页。关闭状态先做会话内，不增加持久化复杂度。

## 与 Review/Practice 的边界

课程库展示课程级概览和入口。Review/Practice 负责任务反馈、错题、需要多练和练习搜索，不在课程库里重复实现这些规则。

## 诚实边界

- 不声称 ASR/OCR provider 已全部接入。
- 不做第三方平台视频爬取。
- 不展示或导出密钥。
- 不把 provider 原始响应、推理内容或内部消息写进课程库。
