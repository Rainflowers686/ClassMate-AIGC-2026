# Stage 8B 图片学习功能 — 用户流程

> 12 步完整用户流程，从拍照到 StudyReport。
> 当前设计阶段，不声称功能已实现。
> 所有"端侧理解"功能明确标注实验性。

---

## 步骤 1：用户拍课件照片

| 项目 | 内容 |
|------|------|
| **页面入口** | 首页 → 资料导入 → 拍照 |
| **用户动作** | 对准课堂 PPT/投影屏幕拍照 |
| **系统行为** | 调用系统相机，拍照后生成 Bitmap，显示预览 |
| **AI 参与点** | 无（此步仅采集） |
| **失败兜底** | 拍照失败 → 提示"请重试或从相册选择" |
| **隐私提示** | "照片仅在设备本地处理，不会上传" |

---

## 步骤 5：App 显示"端侧图像理解实验"标签

| 项目 | 内容 |
|------|------|
| **页面入口** | 图片预览页 → 顶部标签 |
| **用户动作** | 查看标签，了解当前功能状态 |
| **系统行为** | 显示橙色/实验性标签："端侧图像理解 · 实验" |

---

## 步骤 6：生成结构化文本预览

| 项目 | 内容 |
|------|------|
| **页面入口** | 图片预览页 → 点击"开始理解"按钮 |
| **用户动作** | 点击按钮触发端侧分析 |
| **系统行为** | ① RGB 转换 → ② callVit → ③ 多模态 generate → ④ 流式显示文本 |
| **AI 参与点** | **核心 AI 参与**：端侧 BlueLM 3B VIT + 多模态 prompt |
| **失败兜底** | callVit 失败 → "图片处理失败，请尝试云端理解或手动输入" |
| **隐私提示** | "理解过程在设备本地进行，图片数据未上传" |

---

## 步骤 7：用户可编辑

| 项目 | 内容 |
|------|------|
| **页面入口** | 理解结果预览页 |
| **用户动作** | 编辑/修正 AI 生成的文本 |
| **系统行为** | 提供可编辑文本框，保存用户修改 |
| **AI 参与点** | 无（用户修正阶段） |
| **失败兜底** | 无（编辑是纯本地操作） |
| **隐私提示** | "编辑后的内容保存在设备本地" |

---

## 步骤 8：加入资料篮

| 项目 | 内容 |
|------|------|
| **页面入口** | 理解结果页 → "加入资料篮"按钮 |
| **用户动作** | 确认加入 |
| **系统行为** | 将 `[理解文本 + ImageUnderstandingSource + 原图 hash]` 打包，加入 `MaterialBundle` |
| **AI 参与点** | 无 |
| **失败兜底** | 如果原图 hash 计算失败 → 降级为仅保存文本 |
| **隐私提示** | "仅保存理解文本，原图不保存在资料库中" |

---

## 步骤 9：生成 Timeline

| 项目 | 内容 |
|------|------|
| **页面入口** | Timeline 页面自动刷新 |
| **用户动作** | 查看 Timeline |
| **系统行为** | Timeline 中显示新条目：图标 + "[课程名称] 课件截图理解" |
| **AI 参与点** | Timeline 自动从 MaterialBundle 中加载 |
| **失败兜底** | 无理解文本时不显示图片理解条目 |
| **隐私提示** | Timeline 条目不包含原始图片 |

---

## 步骤 10：Ask This Lesson 引用图像理解来源

| 项目 | 内容 |
|------|------|
| **页面入口** | Ask 提问页面 → 来源选择器 |
| **用户动作** | 提问时选择"课件截图理解"作为参考来源 |
| **系统行为** | 将 ImageUnderstandingSource 的文本作为 Ask 的参考上下文 |
| **AI 参与点** | 云端 BlueLM 基于理解文本 + 用户问题生成回答 |
| **失败兜底** | 无理解来源时提供"手动输入参考文本"选项 |
| **隐私提示** | "提问时仅发送理解文本，原始图片不会上传" |

---

## 步骤 11：Quiz 生成

| 项目 | 内容 |
|------|------|
| **页面入口** | Practice → 从资料生成题目 |
| **用户动作** | 选择"课件截图理解"作为题目来源 |

| **AI 参与点** | 云端 BlueLM 基于理解内容出题 |
| **失败兜底** | 理解文本质量不足以出题 → 提示"请补充更多资料后重试" |
| **隐私提示** | 同上 |

---

## 步骤 12：StudyReport 中标注图像来源

| 项目 | 内容 |
|------|------|
| **页面入口** | StudyReport → 学习来源章节 |
| **用户动作** | 阅读报告 |
| **系统行为** | 报告中显示来源列表，图像理解来源标注为"端侧图像理解 · 实验性" |
| **AI 参与点** | 无（报告聚合阶段） |
| **失败兜底** | 无理解来源的报告正常显示其他来源 |
| **隐私提示** | "报告中的图像来源标注不包含原始图片" |
