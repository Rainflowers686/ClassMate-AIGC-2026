# Stage 8A OnDevice — 用户故事地图

> 五类用户角色的端侧模型使用场景与价值分析。
> 每类角色包含：使用流程、端侧模型价值、风险与局限。

---

## 角色一：大学生（在校课堂学习）

### 典型场景

- 在教室用手机录音课堂内容
- 课间/课后用 Ask This Lesson 提问课件相关内容
- 在图书馆自习时生成练习题目
- 导出学习报告作为复习资料

### 使用流程

```
上课 → 录音 ASR 转写 → 课件导入 → Ask 提问
  │         │              │
  ▼         ▼              ▼
云端      云端           优先端侧(简单问答)
                       → 不可用则云端
```

### 端侧模型价值

- **图书馆/自习室场景**：校园 Wi-Fi 不稳定或限速时，端侧模型仍可进行简单问答
- **课间快速提问**：低延迟本地推理，不用等网络响应
- **隐私**：个人学习问题不经过校园网络

### 风险

- 端侧回答质量不如云端 — 复杂专业课问题需走云端
- 模型文件 2–4 GB，可能消耗手机存储空间
- 首次下载模型需要 Wi-Fi

---

## 角色二：考研 / 考公 / 证书备考者

### 典型场景

- 高强度自学，每天数小时刷题
- 使用 Practice 模块生成自适应练习题
- 用 Ask 功能理解错题
- 导出学习报告追踪进度

### 使用流程

```
做题 → 错题记录 → Ask 错题分析 → 自适应新题 → 周报告
  │        │           │              │           │
  ▼        ▼           ▼              ▼           ▼
云端    本地存储      优先端侧        云端        云端
                   (简单概念题)
```

### 端侧模型价值

- **高频简单题**：选择题、填空题的概念解释，端侧可快速响应，节约云端配额
- **通勤/排队碎片时间**：地铁上信号差时仍可刷题和看解析
- **错题隐私**：个人薄弱知识点不经过网络

### 风险

- 主观题/论述题不适合端侧（会降级到云端）
- 连续长时间端侧推理可能导致设备发热
- 错题本同步需要网络（端侧只负责推理，不负责存储同步）

---

## 角色三：在职学习者（业余充电）

### 典型场景

- 晚间或周末碎片化学习
- 导入公司培训材料/行业报告
- 快速生成学习摘要
- 出差途中离线学习

### 使用流程

```
导入资料 → 摘要生成 → 关键概念问答 → 学习笔记
   │           │            │            │
   ▼           ▼            ▼            ▼
 云端       优先端侧      端侧/云端     本地存储
          (简单摘要)   (依复杂度)
```

### 端侧模型价值

- **出差/通勤离线**：飞机上、高铁隧道中可继续学习
- **公司材料隐私**：内部培训资料不经过公有云（端侧处理）
- **快速摘要**：短文档摘要可以本地完成

### 风险

- 长篇报告摘要质量有限（建议复杂文档走云端）
- 不支持 PPT/PDF 内的图片理解（多模态未接入主链路）
- 端侧不支持多语言混合材料的处理

## 角色四：课程老师 / 助教

### 典型场景

- 备课：导入课件 PPT，检查 AI 生成的题目质量
- 课后：查看学生常见问题，调整教学重点
- 出题：生成随堂测验题目

### 使用流程

```
备课 → 课件导入 → AI 生成题目 → 筛选修改 → 发布
  │        │           │           │        │
  ▼        ▼           ▼           ▼        ▼
云端     云端        云端        人工     本地
                              (端侧不做)
```

### 端侧模型价值

- **课件隐私**：未公开的课件/试卷内容不经过公有云
- **本地快速检查**：简单题目质量筛查可以本地完成
- **教室无网**：教室内直接演示端侧问答功能

### 风险

- 端侧**不适合**题目生成（质量要求高，必须走云端）
- 端侧多模态不支持课件图片分析
- 老师角色的端侧模型使用场景较少（主力仍是云端）

---

## 角色五：弱网 / 隐私敏感场景用户

### 典型场景

- 偏远地区网络覆盖差
- 对个人信息高度敏感的群体
- 使用公司/学校 VPN 限制外网访问的环境
- 临时无网环境（地下室、电梯间、地铁）

### 使用流程

```
打开应用 → 端侧诊断 → 确认可用 →
  ├── 提问 → 端侧推理 → 回答
  ├── 做题 → 端侧生成简单题 → 作答
  └── 端侧不可用 → 本地规则兜底
```

### 端侧模型价值

- **最大价值角色**：端侧模型就是为这类场景设计的
- 完全不依赖网络的智能学习体验
- 所有数据不出设备
- 本地规则作为最后兜底

### 风险

- 回答能力和范围受限（3B 模型 + 本地规则的组合）
- 首次使用前必须提前下载模型文件（需要一次性 Wi-Fi）
- 多模态在此角色中价值有限（目前仅诊断）
- 需要用户主动选择"仅端侧"模式（默认是"自动"）

---

## 用户故事优先级矩阵

| 角色 | 端侧使用频率 | 端侧价值 | 风险等级 | 优先级 |
|------|-------------|----------|---------|--------|
| 大学生 | 中 | 中 | 低 | P1 |
| 考研/考公备考者 | 高 | 高 | 中 | **P0** |
| 在职学习者 | 中 | 高 | 中 | P1 |
| 课程老师/助教 | 低 | 低 | 低 | P2 |
| 弱网/隐私场景用户 | 高 | **最高** | 中 | **P0** |

**P0 优先**：考研备考者（高频使用 + 碎片化场景）和弱网/隐私场景用户（端侧核心价值场景）是端侧模型的一级目标用户。

---

## 端侧模型的"不适合"场景

| 场景 | 原因 | 替代方案 |
|------|------|----------|
| 深度论文分析 | 需要长上下文 + 强推理 | 云端 BlueLM |
| 多语言混合翻译 | 3B 模型多语言能力有限 | 云端 BlueLM |
| 数学公式推导 | 需要逐步推理 | 云端 BlueLM |
| 实时语音交互 | 端侧不支持流式音频输入 | 云端 ASR |
| 课件图片 OCR | 多模态未接入主链路 | 云端 OCR |
| 批量导出 | 需要网络同步 | 云端 |
