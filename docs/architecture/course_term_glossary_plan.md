# Course Term Glossary Plan

本文规划课程术语表能力，用于 ASR 热词、OCR 纠错、LLM 提示词术语约束、Quiz 术语一致性。当前为下一阶段方案，不表示已完成 ASR/OCR 接入。

## 1. 为什么需要术语表

### ASR 热词

课堂语音中存在大量专业词、公式读法和缩写。术语表可以作为热词来源，帮助 ASR 更稳定地识别“比值判别法”“楞次定律”“哈斯图”等词。

### OCR 纠错

板书和课件截图可能把公式、英文符号或专业术语识别错。术语表可用于候选纠错，例如把“根植判别法”纠为“根值判别法”。

### LLM 提示词术语约束

分析阶段可以把当前课程术语和别名作为约束，让模型输出知识点标题和微测题时使用一致术语。

### Quiz 术语一致性

微测题、答案解释和复习任务应使用同一套术语，避免同一概念在不同页面被叫成多个名字。

## 2. 数据模型

### CourseTerm

- `id`
- `term`
- `aliases`
- `subject`
- `definition`
- `examples`
- `priority`
- `source`

### 字段说明

- `term`：标准术语。
- `aliases`：别名、英文、缩写、常见误识别写法。
- `subject`：所属科目。
- `definition`：简短定义。
- `examples`：典型出现语境。
- `priority`：热词优先级，核心术语更高。
- `source`：builtin、course_import、teacher_note、user_added。

### TermGlossary

- `subject`
- `terms`
- `version`
- `selectedByUser`
- `generatedFromCourse`

## 4. 后续如何接入

### Import 时选择课程

Import Hub 增加科目选择。选择后加载内置术语表，并允许用户添加本节课专有术语。

### Live 时选择课程

Live Companion 开始前选择科目，ASR 前注入热词，手动片段也可使用术语提示做轻量纠错。

### ASR/OCR 前注入热词

调用 ASR/OCR provider 前，将高优先级术语、别名和课程标题传给对应能力。具体字段取决于官方接口，不猜测协议。

### 分析阶段加入术语提示

CourseAnalyzer 前的材料融合层可把术语表摘要传给分析模块，要求知识点标题、题目和解释使用标准术语。

### 导出报告中保留术语表摘要

Export 可增加“本节课术语表摘要”，列出高优先级术语、别名和简短定义，方便复习。

## 5. 风险与约束

- 术语表不能替代证据链。
- 术语纠错必须可解释，不能随意改写原文证据。
- ASR/OCR 官方接口未确认前，不猜测字段。
- 用户自定义术语不应包含密钥、账号或隐私信息。
